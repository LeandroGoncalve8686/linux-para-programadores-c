1.1 O que é um Sistema Operacional?

Um Sistema Operacional (SO) é o software fundamental que gerencia os recursos de hardware de um computador e fornece serviços para os programas de aplicação. Ele atua como uma camada de abstração entre o hardware físico e o software do usuário.

💡 Analogia

Pense no SO como o gerente de um restaurante: ele não cozinha (isso é o trabalho dos aplicativos), mas organiza a cozinha (hardware), distribui tarefas (CPU), gerencia o estoque (memória) e garante que os pedidos (processos) sejam atendidos na ordem correta.

Os principais componentes de um SO são:

  • Kernel (Núcleo): O coração do sistema. Gerencia processos, memória, dispositivos e chamadas de sistema.
  • Shell: Interface de linha de comando que interpreta comandos do usuário.
  • Sistema de Arquivos: Organiza e armazena dados em dispositivos de armazenamento.
  • Gerenciador de Processos: Controla a execução de programas.
  • Gerenciador de Memória: Aloca e libera memória RAM para processos.
  • Drivers: Permitem que o SO comunique-se com hardware específico.

1.2 O que é Linux?

Linux é um kernel de sistema operacional de código aberto, criado por Linus Torvalds em 1991. O termo "Linux" é frequentemente usado para se referir a sistemas operacionais completos baseados no kernel Linux, que incluem também ferramentas e bibliotecas do projeto GNU.

A história do Linux começou quando Linus, então estudante na Universidade de Helsinque, queria um sistema operacional similar ao UNIX para seu PC, mas não queria pagar pela licença do MINIX. Ele começou a escrever seu próprio kernel.

📜 A Mensagem Original (1991)

"Hello everybody out there using minix — I'm doing a (free) operating system (just a hobby, won't be big and professional like gnu) for 386(486) AT clones."

— Linus Torvalds, comp.os.minix, 25 de agosto de 1991

Linux não é um sistema operacional completo por si só. É um kernel. O que chamamos de "Linux" no dia a dia (Ubuntu, Fedora, Debian) são na verdade distribuições GNU/Linux — combinações do kernel Linux com ferramentas do projeto GNU e outros softwares.

1.3 O Kernel Linux

O kernel é o componente central do sistema operacional. Ele tem acesso total ao hardware e gerencia todos os recursos do sistema. No Linux, o kernel é monolítico com módulos carregáveis — ou seja, a maior parte do código do kernel roda no mesmo espaço de memória, mas drivers e funcionalidades podem ser carregados dinamicamente.

1.3.2 Chamadas de Sistema (System Calls)

Chamadas de sistema são a interface entre o espaço do usuário e o kernel. Quando um programa precisa de um recurso privilegiado (acessar um arquivo, alocar memória, criar um processo), ele faz uma chamada de sistema. O kernel então executa a operação em modo privilegiado e retorna o resultado.

Exemplos de chamadas de sistema importantes:

⚠️ Importante para programadores C

Em C, você não chama as system calls diretamente. Você usa as wrapper functions da glibc (como open(), read(), fork()), que configuram os argumentos e executam a instrução syscall (ou int 0x80 em x86, syscall em x86_64).

1.4 O Projeto GNU

O Projeto GNU foi iniciado por Richard Stallman em 1983 com o objetivo de criar um sistema operacional completamente livre, compatível com UNIX. GNU significa "GNU's Not Unix" — um acrônimo recursivo.

Até 1991, o projeto GNU tinha desenvolvido quase todos os componentes de um sistema operacional: compiladores (GCC), depuradores (GDB), bibliotecas (glibc), shell (bash), editores (Emacs), e utilitários. O que faltava era um kernel. Quando Linus Torvalds lançou o kernel Linux sob licença GPL, a combinação GNU + Linux formou um sistema operacional completo e funcional.

🎯 Por que "GNU/Linux" e não só "Linux"?

Richard Stallman e a Free Software Foundation argumentam que chamar o sistema apenas de "Linux" ignora a contribuição fundamental do projeto GNU. O kernel Linux sozinho não é um sistema operacional utilizável — são as ferramentas GNU que o tornam funcional.

1.5 Shell e Terminal

É fundamental entender a diferença entre terminal, console, shell e TTY — termos frequentemente confundidos.

1.5.1 Terminal

Um terminal (ou terminal emulator) é um programa que emula um dispositivo de terminal físico (como os antigos terminais VT100). Ele fornece uma janela onde você pode interagir com o shell. Exemplos: GNOME Terminal, Konsole, iTerm2, Windows Terminal, xterm.

1.5.2 Console

O console é o dispositivo físico de entrada/saída do sistema. Em servidores, é o monitor e teclado conectados diretamente à máquina. No Linux, os consoles virtuais são acessados com Ctrl+Alt+F1 a F6. O ambiente gráfico geralmente roda no F7 ou F1 (dependendo da distribuição).

1.5.3 Shell

O shell é um programa interpretador de comandos. Ele lê o que você digita, interpreta e executa. O shell padrão no Linux é o Bash (Bourne Again Shell), mas existem alternativas como Zsh, Fish, Ksh, Dash.

1.5.4 TTY

TTY (Teletypewriter) é um conceito herdado dos mainframes. No Linux, um TTY é uma interface de terminal virtual. Cada sessão de login tem um TTY associado. Você pode ver o seu TTY com o comando tty:

$ tty
/dev/pts/0

Os TTYs físicos são /dev/tty1 a /dev/tty6. Os TTYs pseudo (pts) são usados por terminais emulados em ambiente gráfico e conexões SSH.

1.6 Bash — O Shell Padrão

O Bash (Bourne Again Shell) é o shell padrão na maioria das distribuições Linux. É uma reimplementação do Bourne Shell (sh) com recursos adicionais do Korn Shell (ksh) e C Shell (csh).

1.6.1 Recursos do Bash

  • History: Armazena comandos anteriores (arquivo ~/.bash_history)
  • Tab completion: Completa comandos, arquivos e variáveis com Tab
  • Aliases: Atalhos para comandos longos
  • Variables: Variáveis de ambiente e locais
  • Scripts: Programação de automações
  • Job control: Controle de processos em foreground/background
  • Redirection: Redirecionamento de entrada/saída
  • Pipes: Encadeamento de comandos

1.6.2 Arquivos de Configuração do Bash

🔍 Login Shell vs. Non-Login Shell

Um login shell é iniciado quando você faz login no sistema (SSH, console físico). Ele executa /etc/profile e ~/.bash_profile. Um non-login shell é iniciado dentro de uma sessão já logada (abrir um terminal no ambiente gráfico). Ele executa ~/.bashrc.

1.9 systemd — O Sistema de Inicialização

O systemd é o sistema de inicialização e gerenciamento de serviços padrão na maioria das distribuições Linux modernas (Ubuntu, Fedora, Debian, CentOS, Arch). Ele substituiu o antigo SysVinit.

1.9.1 Conceitos do systemd

  • Units: Recursos que o systemd gerencia. Cada unit tem um arquivo de configuração.
  • Services: Processos que o systemd inicia e monitora.
  • Targets: Estados do sistema (equivalentes aos runlevels).
  • Timers: Agendamento de tarefas (alternativa ao cron).
  • Mounts: Pontos de montagem gerenciados pelo systemd.
  • Sockets: Ativação de serviços sob demanda.

1.9.2 Comandos systemd Essenciais

# Verificar status de um serviço
$ systemctl status sshd

# Iniciar um serviço
$ sudo systemctl start nginx

# Parar um serviço
$ sudo systemctl stop nginx

# Habilitar serviço para iniciar no boot
$ sudo systemctl enable nginx

# Desabilitar serviço no boot
$ sudo systemctl disable nginx

# Recarregar configuração de um serviço
$ sudo systemctl reload nginx

# Reiniciar um serviço
$ sudo systemctl restart nginx

# Listar todos os serviços ativos
$ systemctl list-units --type=service --state=active

# Ver logs de um serviço
$ journalctl -u nginx

# Ver logs em tempo real
$ journalctl -f

# Ver logs desde o último boot
$ journalctl -b

1.11 Usuários, Root, UID e GID

Linux é um sistema multiusuário. Cada usuário tem uma identidade única e permissões específicas.

1.11.1 O Usuário Root

O root (UID 0) é o superusuário do sistema. Ele tem acesso irrestrito a todos os arquivos e pode executar qualquer operação. O poder do root é também seu maior perigo — um comando errado pode destruir o sistema.

🚨 Regra de Ouro

NUNCA use o root para tarefas do dia a dia. Use uma conta de usuário regular e eleve privilégios apenas quando necessário com sudo. Isso protege o sistema de erros acidentais e limita o impacto de malware.

1.11.2 UID e GID

# Ver informações de outro usuário

# Ver todos os usuários do sistema

1.12 Permissões de Arquivos

Linux usa um modelo de permissões baseado em três classes de usuários: proprietário (owner), grupo (group), e outros (others). Cada classe pode ter permissões de leitura (r), escrita (w), e execução (x).

1.12.2 Notação Numérica (Octal)

Cada permissão tem um valor numérico:

PermissãoValor

Os valores são somados para cada classe:

rwx = 4+2+1 = 7
r-x = 4+0+1 = 5
r-- = 4+0+0 = 4

chmod 754 arquivo.txt  →  rwxr-xr--

1.12.3 chmod — Alterar Permissões

# Notação simbólica
$ chmod u+x script.sh      # Adiciona execução para o dono
$ chmod g-w arquivo.txt    # Remove escrita do grupo
$ chmod o=r arquivo.txt    # Define leitura apenas para outros
$ chmod a+x programa       # Adiciona execução para todos

# Notação octal
$ chmod 755 script.sh      # rwxr-xr-x (scripts executáveis)
$ chmod 644 arquivo.txt    # rw-r--r-- (arquivos comuns)
$ chmod 700 chave.pem      # rwx------ (chaves privadas)
$ chmod 777 diretorio/     # rwxrwxrwx (EVITE!)
🚨 NUNCA use chmod 777

chmod 777 dá permissão total a todos os usuários. Isso é uma falha de segurança grave. Use o princípio do mínimo privilégio: dê apenas as permissões estritamente necessárias.

1.12.4 chown — Alterar Proprietário

# Mudar apenas o dono
$ sudo chown root arquivo.txt

# Mudar dono e grupo
$ sudo chown usuario:grupo arquivo.txt

# Mudar recursivamente em um diretório
$ sudo chown -R usuario:grupo /var/www/

# Mudar apenas o grupo
$ sudo chown :grupo arquivo.txt

1.12.5 umask — Máscara de Permissões Padrão

O umask define as permissões padrão para novos arquivos e diretórios. É subtraído das permissões máximas.

$ umask
0002

# Permissões máximas: arquivo 666, diretório 777
# Arquivo novo: 666 - 002 = 664 (rw-rw-r--)
# Diretório novo: 777 - 002 = 775 (rwxrwxr-x)

# Alterar umask temporariamente
$ umask 077
# Agora: arquivo 600, diretório 700 (apenas o dono)
🔍 Permissões Especiais

Além de rwx, existem três permissões especiais:

  • SUID (4): Arquivo executa com privilégios do dono. Ex: passwd (rwsr-xr-x)
  • SGID (2): Arquivo executa com privilégios do grupo; diretório herda grupo. Ex: /usr/bin/wall
  • Sticky Bit (1): Apenas o dono pode deletar arquivos no diretório. Ex: /tmp (rwxrwxrwt)
$ chmod 4755 programa    # SUID
$ chmod 2755 diretorio   # SGID
$ chmod 1777 /tmp        # Sticky Bit

1.13 Permissões Avançadas — ACLs

As Access Control Lists (ACLs) permitem definir permissões mais granulares que o modelo tradicional rwx. Com ACLs, você pode dar permissões específicas para usuários e grupos individuais além do dono e grupo principal.

# Ver ACLs de um arquivo
$ getfacl arquivo.txt

# Definir ACL: dar permissão de leitura para um usuário específico
$ setfacl -m u:outro_usuario:r arquivo.txt

# Definir ACL para um grupo
$ setfacl -m m:grupo:rw arquivo.txt

# Remover todas as ACLs
$ setfacl -b arquivo.txt

# Definir ACL padrão para novos arquivos em um diretório
$ setfacl -d -m u:outro_usuario:rwx diretorio/
💡 Quando usar ACLs?

Use ACLs quando precisar de controle de acesso mais fino do que o modelo tradicional permite. Por exemplo: um diretório de projeto onde vários usuários precisam de acesso, mas não fazem parte do mesmo grupo principal.

  5. Crie um diretório projeto, mude o grupo para users com chown, e defina SGID com chmod 2775. Crie um arquivo dentro e verifique o grupo.
  6. Use systemctl list-units --type=service e identifique 5 serviços essenciais do sistema.
  7. Verifique o umask padrão. Altere para 077, crie um arquivo e observe as permissões. Volte ao padrão original.
  8. Instale acl e use setfacl para dar permissão de leitura em um arquivo para outro usuário do sistema.

2.5 rmdir — Remove Directory

Remove diretórios vazios. Para diretórios com conteúdo, use rm -r (com cuidado!).

# Remover diretório vazio
$ rmdir diretorio_vazio

# Remover múltiplos diretórios vazios
$ rmdir dir1 dir2 dir3

# Remover diretório e seus pais se ficarem vazios
$ rmdir -p projeto/2024/junho
# Remove junho/, se 2024/ ficar vazio remove também, etc.

# Remover diretórios vazios recursivamente
$ find . -type d -empty -delete

# Verificar antes de remover
$ rmdir -v diretorio

🚨 rmdir vs. rm -r

rmdir só remove diretórios vazios — é seguro. rm -r remove diretórios e TODO o conteúdo. Use rm -ri (interativo) ou rm -rI (pergunta uma vez para muitos arquivos) para evitar acidentes.

2.6 touch — Change File Timestamps

O touch é usado para criar arquivos vazios ou atualizar timestamps. É mais poderoso do que parece.

# Criar arquivo vazio
$ touch novo_arquivo.txt

# Atualizar timestamp de acesso e modificação para agora
$ touch arquivo_existente.txt

# Atualizar apenas timestamp de acesso
$ touch -a arquivo.txt

# Atualizar apenas timestamp de modificação
$ touch -m arquivo.txt

# Não criar arquivo se não existir
$ touch -c arquivo_inexistente.txt

# Usar timestamp de outro arquivo como referência
$ touch -r arquivo_referencia.txt arquivo_alvo.txt

# Definir timestamp específico
$ touch -t 202406141200.00 arquivo.txt
# Formato: [[CC]YY]MMDDhhmm[.ss]
# 2024-06-14 12:00:00

# Definir data com string legível
$ touch -d "2024-06-14 12:00:00" arquivo.txt
$ touch -d "yesterday" arquivo.txt
$ touch -d "2 days ago" arquivo.txt
$ touch -d "next Monday" arquivo.txt

# Criar múltiplos arquivos
$ touch arquivo{1..10}.txt
# Cria: arquivo1.txt, arquivo2.txt, ..., arquivo10.txt

# Criar arquivo com espaço no nome (use aspas)
$ touch "meu arquivo.txt"

# Criar arquivo oculto
$ touch .config

2.7 cp — Copy Files and Directories

Copia arquivos e diretórios. Um dos comandos mais perigosos se usado sem atenção.

# Copiar arquivo
$ cp origem.txt destino.txt

# Copiar para diretório
$ cp arquivo.txt /backup/

# Copiar múltiplos arquivos
$ cp arquivo1.txt arquivo2.txt /destino/

# Copiar com preservação de atributos
$ cp -p arquivo.txt backup/
# Preserva: modo, ownership, timestamps

# Copiar recursivamente (diretórios)
$ cp -r diretorio/ /backup/

# Copiar recursivamente com preservação completa
$ cp -a diretorio/ /backup/
# Equivalente a -dR --preserve=all

# Copiar com verificação de sobrescrita (interativo)
$ cp -i arquivo.txt /destino/

# Forçar sobrescrita sem perguntar
$ cp -f arquivo.txt /destino/

# Copiar apenas se o arquivo for mais novo
$ cp -u arquivo.txt /destino/

# Copiar com barra de progresso (pv)
$ cp arquivo_grande.iso /destino/ & pv -pteba arquivo_grande.iso > /destino/arquivo_grande.iso

# Copiar com backup automático
$ cp --backup=numbered arquivo.txt /destino/
# Cria: arquivo.txt.~1~, arquivo.txt.~2~, etc.

# Copiar links simbólicos como links (não seguir)
$ cp -d link_simbolico /destino/

# Criar link hard em vez de copiar
$ cp -l arquivo.txt /destino/

# Copiar com sparse (arquivos esparsos)
$ cp --sparse=always arquivo.img /destino/
🚨 Cuidado com cp e diretórios

cp -r copia recursivamente. Se o destino já existe e é um diretório, o conteúdo é copiado dentro dele. Se não existe, cria o diretório. Verifique sempre com ls antes.

2.8 mv — Move/Rename Files

Move ou renomeia arquivos e diretórios. No Linux, renomear e mover são a mesma operação (ambos alteram o path).

# Renomear arquivo
$ mv antigo.txt novo.txt

# Mover arquivo para diretório
$ mv arquivo.txt /destino/

# Mover múltiplos arquivos
$ mv *.txt /backup/

# Mover e sobrescrever sem perguntar
$ mv -f arquivo.txt /destino/

# Mover com confirmação (interativo)
$ mv -i arquivo.txt /destino/

# Mover apenas se o destino for mais antigo
$ mv -u arquivo.txt /destino/

# Mover com backup
$ mv --backup=numbered arquivo.txt /destino/

# Mover diretório inteiro
$ mv diretorio/ /novo/local/

# Renomear com substituição de padrão (bash)
$ for f in *.txt; do mv "$f" "${f%.txt}.bak"; done

# Renomear múltiplos arquivos com rename (perl)
$ rename 's/\.txt$/.bak/' *.txt

# Mover arquivos modificados hoje
$ find . -mtime -1 -type f -exec mv {} /backup/hoje/ \;
💡 mv é atômico para arquivos no mesmo filesystem

Quando mv move um arquivo dentro do mesmo filesystem, ele apenas atualiza os metadados no inode — os dados do arquivo não são copiados. Isso é instantâneo e atômico, mesmo para arquivos gigantes.

2.9 rm — Remove Files

Remove arquivos. PERMANENTEMENTE. Não há lixeira no terminal. Use com extrema cautela.

# Remover arquivo
$ rm arquivo.txt

# Remover múltiplos arquivos
$ rm arquivo1.txt arquivo2.txt

# Remover com confirmação
$ rm -i arquivo.txt

# Remover sem confirmação (padrão para root!)
$ rm -f arquivo.txt

# Remover diretório e conteúdo recursivamente
$ rm -r diretorio/

# Remover diretório vazio
$ rm -d diretorio_vazio/
# Equivalente a rmdir

# Remover recursivamente e forçado (MUITO CUIDADO!)
$ rm -rf diretorio/

# Remover arquivos com nomes estranhos
$ rm -- -arquivo_com_hifen_no_inicio
$ rm ./-arquivo_com_hifen

# Remover exceto um arquivo
$ rm !(arquivo_importante.txt)
# Requer: shopt -s extglob

# Remover arquivos antigos
$ find /tmp -type f -mtime +7 -delete

# Remover com log
$ rm -v arquivo.txt
removed 'arquivo.txt'

# Simular remoção (não remove, apenas mostra o que faria)
# Use echo ou rm -i combinado com find
$ find . -name "*.tmp" -exec echo rm {} \;
🚨 O Comando Mais Perigoso do Linux

rm -rf / apaga TODO o sistema. Algumas distribuições modernas têm proteção (rm --preserve-root é padrão), mas rm -rf /* ou rm -rf / --no-preserve-root ainda funcionam. Sempre verifique o caminho antes de executar rm -rf.

Regra: Nunca use rm -rf com variáveis não verificadas ou com wildcards sem testar primeiro.

2.10 cat — Concatenate and Print

O cat concatena e exibe arquivos. É frequentemente usado para visualizar arquivos pequenos, mas tem usos mais avançados.

# Exibir conteúdo de arquivo
$ cat arquivo.txt

# Concatenar múltiplos arquivos
$ cat arquivo1.txt arquivo2.txt > combinado.txt

# Concatenar com numeração de linhas
$ cat -n arquivo.txt

# Numeração apenas para linhas não-vazias
$ cat -b arquivo.txt

# Mostrar caracteres especiais (tabs, fim de linha)
$ cat -A arquivo.txt
# ^I = tab, $ = fim de linha

# Suprimir linhas repetidas em branco
$ cat -s arquivo.txt

# Criar arquivo com heredoc
$ cat > arquivo.txt << 'EOF'

# Criar arquivo com conteúdo inline
$ cat > config.ini < arquivo.txt
# Equivalente a: > arquivo.txt
⚠️ Useless Use of Cat (UUOC)

Usar cat arquivo | comando quando comando arquivo funciona é considerado um anti-pattern. O cat cria um processo desnecessário. Prefira grep "padrao" arquivo.txt em vez de cat arquivo.txt | grep "padrao".

O less é um paginador que permite navegar por arquivos grandes de forma interativa. É o sucessor do more e muito mais poderoso.

Exibe as últimas linhas de um arquivo. Essencial para monitorar logs em tempo real.

$ tail arquivo.txt

$ tail -n 20 arquivo.txt

2.17 file — Determine File Type

# Identificar tipo de arquivo
$ file arquivo.txt
arquivo.txt: ASCII text

$ file programa
programa: ELF 64-bit LSB executable, x86-64, version 1 (SYSV)

$ file imagem.png
imagem.png: PNG image data, 800 x 600, 8-bit/color RGBA

$ file *

# Identificar com mais detalhes
$ file -i arquivo.txt
arquivo.txt: text/plain; charset=us-ascii

# Ver magic number em hexadecimal
$ file -k arquivo_desconhecido

# Listar todos os tipos de arquivo conhecidos
$ file -l

# Identificar arquivo dentro de arquivo compactado
$ file -z arquivo.tar.gz

# Verificar arquivo de core dump

💡 Como file funciona?

O file usa um banco de dados de "magic numbers" (bytes iniciais característicos) localizado em /usr/share/file/magic. Por exemplo, arquivos ELF começam com 0x7F 'E' 'L' 'F', arquivos PNG com 0x89 'P' 'N' 'G'.

grep, find, sed, awk e as ferramentas que transformam dados em informação.

3.1 grep — Global Regular Expression Print

O grep busca padrões em texto usando expressões regulares. É uma das ferramentas mais poderosas do terminal.

3.1.1 Uso Básico

# Buscar palavra em arquivo
$ grep "erro" log.txt

# Buscar em múltiplos arquivos
$ grep "erro" *.log

# Buscar recursivamente em diretórios
$ grep -r "erro" /var/log/

# Buscar ignorando case
$ grep -i "erro" log.txt
# Encontra: erro, ERRO, Erro, eRrO

# Inverter match (linhas que NÃO contêm o padrão)
$ grep -v "sucesso" log.txt

# Mostrar número da linha
$ grep -n "erro" log.txt

# Mostrar apenas contagem
$ grep -c "erro" log.txt

# Mostrar nome do arquivo (com múltiplos arquivos)
$ grep -l "erro" *.log

# Mostrar linhas que NÃO contêm o padrão
$ grep -L "erro" *.log

# Mostrar contexto (linhas ao redor)
$ grep -C 2 "erro" log.txt   # 2 linhas antes e depois
$ grep -B 2 "erro" log.txt   # 2 linhas antes
$ grep -A 2 "erro" log.txt   # 2 linhas depois

3.1.2 Expressões Regulares com grep

# Buscar no início da linha (^)
$ grep "^erro" log.txt

# Buscar no final da linha ($)
$ grep "erro$" log.txt

# Qualquer caractere (.)
$ grep "e.r.o" log.txt
# Encontra: erro, e1r2o, e r o

# Conjunto de caracteres ([])
$ grep "[Ee]rro" log.txt
# Encontra: erro, Erro

# Range de caracteres
$ grep "[a-z]erro" log.txt
# Encontra: aerro, berro, ..., zerro

# Negar conjunto
$ grep "[^0-9]erro" log.txt
# Encontra erro precedido por algo que nao eh numero

# Quantificadores
$ grep "erro*" log.txt    # 0 ou mais 'r'
$ grep "erro+" log.txt    # 1 ou mais 'r' (grep -E)
$ grep "erro?" log.txt    # 0 ou 1 'r' (grep -E)
$ grep "erro{2,3}" log.txt # 2 a 3 'r' (grep -E)

# Alternativa (OR)
$ grep -E "erro|falha" log.txt

# Grupos
$ grep -E "(erro|falha) critica" log.txt

# Palavra completa (\b)
$ grep -w "erro" log.txt
# Nao encontra "errores", apenas "erro" isolado

# grep estendido (ERE)
$ grep -E "padrao+" arquivo.txt
# ou
$ egrep "padrao+" arquivo.txt

3.1.3 grep Avancado

# Buscar apenas palavras completas
$ grep -w "root" /etc/passwd

# Limitar numero de matches
$ grep -m 5 "erro" log.txt

# Buscar em arquivo binario (tratar como texto)
$ grep -a "texto" arquivo.bin

# Mostrar posicao do match
$ grep -o "erro" log.txt
# Apenas o texto matchado, nao a linha inteira

# Buscar com arquivo de padroes
$ grep -f padroes.txt log.txt
# padroes.txt contem um padrao por linha

# Excluir arquivos/diretorios na busca recursiva
$ grep -r "erro" /var/log/ --exclude="*.gz"
$ grep -r "erro" /var/log/ --exclude-dir="archive"

# Colorir output
$ grep --color=auto "erro" log.txt

# Contar ocorrencias por arquivo
$ grep -c "erro" *.log

# Mostrar apenas arquivos SEM match
$ grep -L "erro" *.log

# Usar com process substitution
$ grep "erro" <(dmesg)

# grep com Perl regex (PCRE)
$ grep -P "\d{3}-\d{2}-\d{4}" arquivo.txt
# Busca SSN: 123-45-6789
💡 grep vs. egrep vs. fgrep
  • grep: Basico (BRE - Basic Regular Expressions)
  • egrep ou grep -E: Estendido (ERE - Extended Regular Expressions)
  • fgrep ou grep -F: Fixed strings (sem regex, busca literal)
  • grep -P: Perl Compatible Regular Expressions (PCRE)

3.2 find — Search for Files

O find busca arquivos e diretorios no sistema de arquivos. E extremamente poderoso e flexivel.

3.2.1 Busca Basica

# Buscar por nome
$ find /home -name "*.txt"

# Buscar ignorando case
$ find /home -iname "*.TXT"

# Buscar por tipo
$ find /home -type f          # arquivos regulares
$ find /home -type d          # diretorios
$ find /home -type l          # links simbolicos
$ find /home -type b          # dispositivos de bloco
$ find /home -type c          # dispositivos de caractere
$ find /home -type p          # named pipes
$ find /home -type s          # sockets

# Buscar por tamanho
$ find /home -size +100M      # maior que 100MB
$ find /home -size -1k        # menor que 1KB
$ find /home -size 10M        # exatamente 10MB
$ find /home -size +1G -size -5G  # entre 1GB e 5GB

# Buscar por tempo de modificacao
$ find /home -mtime -1        # modificado nas ultimas 24h
$ find /home -mtime +7        # modificado ha mais de 7 dias
$ find /home -mtime 1         # modificado entre 24h e 48h atras

# Buscar por tempo de acesso
$ find /home -atime +30       # nao acessado ha 30+ dias

# Buscar por tempo de mudanca (ctime)
$ find /home -ctime -1

# Buscar por minutos (mais preciso)
$ find /home -mmin -60        # modificado na ultima hora

3.2.2 Operadores Logicos

# AND (padrao)
$ find /home -name "*.txt" -size +1M

# OR
$ find /home \( -name "*.txt" -o -name "*.log" \)

# NOT
$ find /home ! -name "*.txt"

# Combinados
$ find /home -type f \( -name "*.tmp" -o -name "*.bak" \) -size +10M

# Precedencia (use parenteses)
$ find /home \( -type f -a -name "*.log" \) -o -type d

3.2.3 Acoes com find

# Executar comando para cada arquivo encontrado
$ find /home -name "*.log" -exec ls -lh {} \;

# Executar com multiplos arquivos de uma vez (mais eficiente)
$ find /home -name "*.log" -exec ls -lh {} +

# Deletar arquivos encontrados
$ find /tmp -name "*.tmp" -type f -delete

# Ou com exec
$ find /tmp -name "*.tmp" -type f -exec rm {} \;

# Copiar arquivos encontrados
$ find /home -name "*.conf" -exec cp {} /backup/ \;

# Mover arquivos
$ find /home -name "*.old" -exec mv {} /arquivo/ \;

# Executar script
$ find /home -name "*.sh" -exec bash {} \;

# Chmod em arquivos encontrados
$ find /home -type f -exec chmod 644 {} \;
$ find /home -type d -exec chmod 755 {} \;

# Usar -ok para confirmacao interativa
$ find /home -name "*.txt" -ok rm {} \;

3.2.4 find Avancado

# Buscar por permissoes
$ find /home -perm 644
$ find /home -perm /u=s       # SUID
$ find /home -perm -g=w       # grupo tem escrita

# Buscar por dono
$ find /home -user root
$ find /home -group sudo

# Buscar por inode
$ find / -inum 123456

# Buscar links para um arquivo especifico
$ find / -samefile /path/to/file

# Buscar arquivos vazios
$ find /home -type f -empty

# Buscar diretorios vazios
$ find /home -type d -empty

# Limitar profundidade
$ find /home -maxdepth 2 -name "*.txt"
$ find /home -mindepth 2 -name "*.txt"

# Nao seguir links simbolicos (padrao)
$ find /home -name "*.txt"

# Seguir links simbolicos
$ find /home -follow -name "*.txt"

# Buscar em multiplos diretorios
$ find /home /var /tmp -name "*.log"

# Excluir diretorios da busca
$ find /home -path "*/node_modules" -prune -o -name "*.js" -print

# Usar -printf para formatar saida
$ find /home -type f -printf "%p %s %TY-%Tm-%Td\n"
# path, size, data

# Buscar com regex (caminho completo)
$ find /home -regex ".*\.txt$"

# Buscar com regex ignorando case
$ find /home -iregex ".*\.TXT$"
✅ find + xargs eh mais eficiente que find -exec

Para processar muitos arquivos, find ... | xargs comando eh mais eficiente que find ... -exec comando {} \; porque xargs passa multiplos arquivos de uma vez. Use find ... -print0 | xargs -0 comando para nomes com espacos.

3.3 locate — Find Files by Name

O locate busca em um banco de dados pre-construido, sendo muito mais rapido que find para buscas por nome. O banco de dados eh atualizado pelo updatedb.

# Instalar mlocate
$ sudo apt install mlocate

# Buscar arquivo por nome
$ locate arquivo.txt

# Buscar ignorando case
$ locate -i arquivo.txt

# Mostrar apenas N resultados
$ locate -n 10 arquivo.txt

# Mostrar arquivos existentes (verifica se existem)
$ locate -e arquivo.txt

# Atualizar banco de dados manualmente
$ sudo updatedb

# Buscar com regex
$ locate -r "\.conf$"

# Buscar arquivos de um usuario especifico
$ locate -u usuario arquivo.txt

# Estatisticas do banco de dados
$ locate -S
# Mostra: arquivos, diretorios, tamanho do DB, etc.

# Buscar apenas arquivos (nao diretorios)
$ locate -r "\.txt$"

# Excluir padroes do updatedb
# Edite /etc/updatedb.conf para excluir /tmp, /proc, etc.
⚠️ locate nao encontra arquivos recem-criados

O banco de dados do locate eh atualizado periodicamente (geralmente uma vez por dia via cron). Arquivos criados apos a ultima atualizacao nao aparecerao. Use find para buscas em tempo real.

3.6 sort — Sort Lines of Text

O sort ordena linhas de texto. Suporta ordenacao numerica, por data, por campo, e mais.

# Ordenar alfabeticamente
$ sort arquivo.txt

# Ordenar inversamente
$ sort -r arquivo.txt

# Ordenar numericamente
$ sort -n numeros.txt

# Ordenar por tamanho (human-readable)
$ sort -h tamanhos.txt

# Ordenar ignorando case
$ sort -f arquivo.txt

# Ordenar removendo duplicatas
$ sort -u arquivo.txt

# Ordenar por campo especifico
$ sort -k 3 arquivo.txt     # 3o campo (padrao: tab/space)
$ sort -k 2,2n arquivo.txt  # 2o campo, numerico
$ sort -t: -k 3n /etc/passwd # delimitador ':', 3o campo numerico

# Ordenar por multiplos campos
$ sort -t: -k 3n -k 1,1 /etc/passwd

# Ordenar aleatoriamente (shuffle)
$ sort -R arquivo.txt

# Ordenar e verificar se ja esta ordenado
$ sort -c arquivo.txt

# Ordenar com locale especifico
$ LC_ALL=C sort arquivo.txt

# Ordenar arquivo grande (merge sort externo)
$ sort -S 50% arquivo_grande.txt
# Usa 50% da RAM

# Ordenar saida de comando
$ ps aux | sort -k 3 -n -r  # por CPU%, numerico, reverso
$ du -h /home | sort -h -r  # por tamanho, maior primeiro

# Top 10 processos por uso de memoria

# Ordenar arquivos por tamanho

# Ordenar IPs numericamente

3.7 uniq — Remove Duplicate Lines

O uniq remove ou conta linhas duplicadas adjacentes. Importante: so remove duplicatas adjacentes — use sort | uniq para remover todas.

# Remover duplicatas adjacentes
$ uniq arquivo.txt

# Contar ocorrencias
$ uniq -c arquivo.txt

$ uniq -d arquivo.txt

$ uniq -u arquivo.txt

$ uniq -w 10 arquivo.txt

$ uniq -i arquivo.txt

# ou mais eficiente:

# Contar ocorrencias de cada linha (histograma)

3.9 cut — Remove Sections from Lines

O cut extrai colunas/bytes de cada linha. Util para processar arquivos delimitados.

$ cut -f 1,3 arquivo.txt    # 1o e 3o campos

# Extrair com delimitador customizado
$ cut -d: -f 1 /etc/passwd    # usuarios

$ cut -b 1-10 arquivo.txt

# Extrair por caracteres
$ cut -c 1-10 arquivo.txt

# Extrair complemento (tudo EXCETO)
$ cut --complement -d: -f 2 /etc/passwd
# Todos os campos EXCETO o 2o

# Combinar com outros comandos
$ ps aux | cut -c 1-10      # primeiros 10 chars
$ df -h | cut -d' ' -f 1    # filesystems

# Listar apenas nomes de usuarios

3.11 tr — Translate or Delete Characters

O tr traduz, comprime ou deleta caracteres. Opera apenas em stdin (nao aceita arquivo como argumento).

# Converter maiusculas para minusculas
$ echo "HELLO" | tr 'A-Z' 'a-z'
hello

# Deletar caracteres
$ echo "hello 123 world" | tr -d '0-9'
hello  world

# Deletar espacos
$ echo "hello world" | tr -d ' '
helloworld

# Comprimir caracteres repetidos
$ echo "hello    world" | tr -s ' '
hello world

# Substituir espacos por tabs
$ echo "col1 col2 col3" | tr ' ' '\t'

# Substituir newlines por espacos
$ cat arquivo.txt | tr '\n' ' '

# Remover caracteres nao imprimiveis
$ tr -cd '[:print:]\n' < arquivo.txt

# Usar classes de caracteres
$ echo "hello123" | tr '[:lower:]' '[:upper:]'
$ echo "hello 123" | tr -d '[:digit:]'
$ echo "hello!!!" | tr -d '[:punct:]'

uryyb

# Criar senha aleatoria
$ tr -dc 'A-Za-z0-9' < /dev/urandom | head -c 16

# Contar caracteres unicos
$ echo "hello" | tr 'a-z' '\n' | sort | uniq -c

3.12 sed — Stream Editor

O sed eh um editor de streams nao-interativo. Pode substituir, deletar, inserir, e transformar texto linha por linha.

3.12.1 Substituicao (s///)

# Substituir primeira ocorrencia por linha
$ sed 's/erro/ERRO/' log.txt

# Substituir TODAS as ocorrencias (global)
$ sed 's/erro/ERRO/g' log.txt

# Substituir apenas No ocorrencia
$ sed 's/erro/ERRO/2' log.txt   # 2a ocorrencia
$ sed 's/erro/ERRO/2g' log.txt  # da 2a em diante

# Substituir com regex
$ sed 's/erro[0-9]*/ERRO/g' log.txt

# Usar delimitador diferente (uteis para paths)
$ sed 's|/home/usuario|/home/novo|g' arquivo.txt
$ sed 's#/usr/local#/opt#g' arquivo.txt

# Substituir e salvar no mesmo arquivo (in-place)
$ sed -i 's/erro/ERRO/g' log.txt

# Backup antes de editar
$ sed -i.bak 's/erro/ERRO/g' log.txt
# Cria log.txt.bak

# Substituir com referencia ao match
$ sed 's/\(erro\)/[\1]/g' log.txt
# [erro]

# Substituir com grupos numerados
$ echo "2024-06-14" | sed 's/\([0-9]*\)-\([0-9]*\)-\([0-9]*\)/\3\/\2\/\1/'
14/06/2024

3.12.2 Deletar Linhas

# Deletar linha 5
$ sed '5d' arquivo.txt

# Deletar linhas 5 a 10
$ sed '5,10d' arquivo.txt

# Deletar linhas que contem padrao
$ sed '/erro/d' log.txt

# Deletar linhas que NAO contem padrao
$ sed '/erro/!d' log.txt

# Deletar linhas em branco
$ sed '/^$/d' arquivo.txt

# Deletar linhas comentadas
$ sed '/^#/d' arquivo.txt

# Deletar primeira e ultima linha
$ sed '1d;$d' arquivo.txt

# Deletar a ultima linha
$ sed '$d' arquivo.txt

3.12.3 Inserir e Adicionar

# Inserir antes da linha 1
$ sed '1i\# Header' arquivo.txt

# Adicionar depois da linha 1
$ sed '1a\Nova linha' arquivo.txt

# Inserir antes de linhas com padrao

$ sed '$a\Fim do arquivo' arquivo.txt

# Substituir linha inteira

3.13 awk — Pattern Scanning and Processing

O awk eh uma linguagem de programacao completa para processamento de texto. Pode fazer calculos, condicionais, loops, e manipular dados estruturados.

3.13.1 Uso Basico

# Imprimir campo especifico (delimitador padrao: espaco/tab)
$ awk '{print $1}' arquivo.txt

# Imprimir multiplos campos
$ awk '{print $1, $3}' arquivo.txt

# Usar delimitador customizado
$ awk -F: '{print $1}' /etc/passwd

# Imprimir com formato
$ awk '{print "Usuario:", $1, "- UID:", $3}' /etc/passwd

# Imprimir numero de campos
$ awk '{print NF}' arquivo.txt

# Imprimir linha inteira
$ awk '{print $0}' arquivo.txt

# Imprimir ultimo campo
$ awk '{print $NF}' arquivo.txt

# Imprimir com separador de saida
$ awk -F: -v OFS=' | ' '{print $1, $6}' /etc/passwd

3.13.2 Padroes e Condicoes

# Filtrar linhas com padrao
$ awk '/erro/' log.txt

# Filtrar por valor de campo
$ awk '$3 > 100' arquivo.txt

# Filtrar com regex no campo
$ awk '$2 ~ /^[0-9]/' arquivo.txt

# Multiplas condicoes
$ awk '$3 > 100 && $5 == "ativo"' arquivo.txt

# Usar BEGIN e END
$ awk 'BEGIN {print "Inicio"} {print $0} END {print "Fim"}' arquivo.txt

# Contar linhas
$ awk 'END {print NR}' arquivo.txt

# Somar valores de campo
$ awk '{soma += $3} END {print "Total:", soma}' arquivo.txt

# Media
$ awk '{soma += $3; count++} END {print "Media:", soma/count}' arquivo.txt

# Maximo e minimo
$ awk 'NR==1 {max=$3; min=$3} $3>max {max=$3} $3<min {min=$3} END {print "Max:", max, "Min:", min}' arquivo.txt

3.13.3 awk Avancado

# Usar variaveis
$ awk -v limite=100 '$3 > limite' arquivo.txt

# Usar arrays
$ awk '{count[$1]++} END {for (u in count) print u, count[u]}' log.txt

# Ordenar saida (pipe para sort)
$ awk '{count[$1]++} END {for (u in count) print count[u], u}' log.txt | sort -n -r

# Formatar saida com printf
$ awk '{printf "%-15s %10d\n", $1, $3}' arquivo.txt

# Processar CSV
$ awk -F, '{print $2}' dados.csv

# Substituir dentro do awk
$ awk '{gsub(/erro/, "ERRO"); print}' log.txt

# Substr (extrair parte do campo)
$ awk '{print substr($1, 1, 3)}' arquivo.txt

# Length (tamanho da linha/campo)
$ awk '{print length($0)}' arquivo.txt

# Split (dividir campo)
$ awk '{split($1, a, "."); print a[1]}' arquivo.txt

# Match (encontrar posicao do regex)
$ awk '{if (match($0, /[0-9]+/)) print substr($0, RSTART, RLENGTH)}' arquivo.txt

# Ler multiplos arquivos
$ awk '{print FILENAME, $0}' arquivo1.txt arquivo2.txt

# Processar registros multilinha
$ awk 'BEGIN {RS=""; FS="\n"} {print $1}' arquivo.txt

3.13.4 Exemplos Praticos de awk

# Top 5 processos por memoria
$ ps aux | awk '{print $4, $11}' | sort -n -r | head -5

# Soma de uso de disco por usuario

# Extrair IPs de log

# Relatorio de tamanho de diretorios

# Filtrar /etc/passwd: usuarios com UID >= 1000
$ awk -F: '$3 >= 1000 {print $1, $6}' /etc/passwd

# Calcular tempo total de processos
$ ps -eo time | awk -F: '{sum += $1*3600 + $2*60 + $3} END {print sum "s"}'

📚 Volume 4 — Pipes e Redirecionamentos

A alma do Unix: composicao de programas simples para criar solucoes poderosas.

4.1 Os Tres Fluxos de Dados

Todo processo no Linux tem tres fluxos de dados padrao:

┌─────────────────────────────────────┐ │ PROCESSO │ │ │ │ stdin (0) ─────→ programa │ │ ↑ │ │ teclado / pipe / arquivo │ │ │ │ stdout (1) ─────→ tela / pipe │ │ ↓ │ │ terminal / arquivo │ │ │ │ stderr (2) ─────→ tela / arquivo │ │ ↓ │ │ terminal / arquivo │ └─────────────────────────────────────┘
FluxoDescritorDescricaoUso Padrao
stdin0Entrada padraoTeclado
stdout1Saida padraoTerminal
stderr2Saida de erroTerminal

Esses fluxos sao implementados como arquivos no Linux. O kernel abre tres descritores de arquivo (file descriptors) para cada processo: 0 para stdin, 1 para stdout, e 2 para stderr.

4.2 Redirecionamento de Saida (stdout) — > e >>

4.2.1 Sobrescrever Arquivo (>)

# Salvar saida em arquivo (SOBRESCREVE se existir)
$ echo "Hello" > arquivo.txt

# Salvar saida de comando
$ ls -la > lista.txt

# Salvar saida de programa
$ ./programa > saida.log

# Redirecionar para /dev/null (descartar)
$ ./programa > /dev/null
🚨 > SOBRESCREVE sem aviso!

Se arquivo.txt ja existir, > o apaga e cria um novo. Nao ha lixeira. Use >> para anexar, ou set -o noclobber no bash para prevenir sobrescrita acidental.

4.2.2 Anexar a Arquivo (>>)

# Anexar saida ao final do arquivo
$ echo "Nova linha" >> arquivo.txt

# Acumular logs
$ ./programa >> log.txt

# Adicionar ao .bashrc
$ echo 'alias ll="ls -la"' >> ~/.bashrc

4.2.3 noclobber — Prevenir Sobrescrita

# Habilitar protecao
$ set -o noclobber

# Agora isso falha se arquivo existir
$ echo "teste" > arquivo_existente.txt
bash: arquivo_existente.txt: cannot overwrite existing file

# Forcar sobrescrita mesmo com noclobber
$ echo "teste" >| arquivo_existente.txt

# Desabilitar protecao
$ set +o noclobber

4.3 Redirecionamento de Entrada (stdin) — <

# Usar arquivo como entrada
$ sort < nomes.txt

# Contar palavras de arquivo
$ wc -w < texto.txt

# Usar heredoc como entrada
$ cat << EOF
Linha 1
Linha 2
EOF

# Usar here-string
$ tr 'a-z' 'A-Z' <<< "hello"
HELLO

# Redirecionar entrada e saida
$ sort < nomes.txt > nomes_ordenados.txt

4.3.1 Heredoc (<<)

# Heredoc basico
$ cat << EOF
Este eh um texto
com multiplas linhas
EOF

# Heredoc sem expansao de variaveis (aspas no delimitador)
$ cat << 'EOF'
$HOME nao sera expandido
EOF

# Heredoc com tab stripping (<<-)
$ cat <<- EOF
	Esta linha tera o tab removido
		Esta tambem
EOF

# Heredoc em script SQL
$ mysql -u root << 'SQL'

SQL

4.3.2 Here-string (<<<)

# Passar string como stdin
$ grep "padrao" <<< "texto com padrao aqui"

# Usar com comando que espera arquivo
$ base64 <<< "senha123"

# Processar variavel
$ texto="hello world"
$ tr 'a-z' 'A-Z' <<< "$texto"

4.4 Redirecionamento de Erro (stderr) — 2> e 2>>

# Redirecionar apenas erros para arquivo
$ ./programa 2> erros.log

# Redirecionar erros e descartar
$ ./programa 2> /dev/null

# Acumular erros
$ ./programa 2>> erros.log

# Separar stdout e stderr
$ ./programa > saida.log 2> erros.log

# Juntar stdout e stderr no mesmo arquivo
$ ./programa > tudo.log 2>&1

# Ordem importa! Isso funciona:
$ ./programa > tudo.log 2>&1

# Isso NAO funciona como esperado (stderr vai para tela):
$ ./programa 2>&1 > tudo.log

4.4.1 Combinando stdout e stderr

# Ambos para o mesmo arquivo
$ comando > arquivo.log 2>&1

# Ambos para /dev/null
$ comando > /dev/null 2>&1
# Atalho em bash 4+: &> /dev/null

# stdout para arquivo, stderr para outro
$ comando > saida.log 2> erros.log

# stdout para pipe, stderr para arquivo
$ comando 2> erros.log | outro_comando

# stderr para stdout, depois pipe
$ comando 2>&1 | grep "erro"
💡 Por que 2>&1 e nao 2>1?

2>1 redirecionaria stderr para um arquivo chamado "1". 2>&1 significa "redirecione stderr (2) para onde stdout (1) esta apontando". O & indica que o numero seguinte eh um descritor de arquivo, nao um nome de arquivo.

4.5 Pipe (|) — Conectar Comandos

O pipe conecta o stdout de um comando ao stdin de outro. E a caracteristica mais poderosa do shell Unix.

# Pipeline basica
$ ls -la | grep ".txt"

# Multiplos pipes
$ cat log.txt | grep "erro" | sort | uniq -c | sort -n -r

# Contar processos
$ ps aux | wc -l

# Top 10 maiores diretorios
$ du -h /home | sort -h -r | head -10

# Filtrar e processar
$ cat /etc/passwd | grep "/bin/bash" | cut -d: -f1

# Pipeline com awk
$ ps aux | awk '{print $2, $11}' | grep "firefox"

# Pipeline com sed
$ ifconfig | grep "inet " | sed 's/inet //' | awk '{print $1}'

4.5.1 Pipe com stderr

# Pipe apenas stdout
$ comando | grep "padrao"

# Pipe stdout e stderr juntos
$ comando 2>&1 | grep "padrao"

# Pipe apenas stderr (troca stdout e stderr)
$ comando 3>&1 1>&2 2>&3 | grep "erro"

4.5.2 Process Substitution (<())

# Comparar saida de dois comandos
$ diff <(ls dir1) <(ls dir2)

# Usar saida de comando como arquivo
$ grep "padrao" <(cat arquivo1.txt arquivo2.txt)

# Combinar multiplas saidas
$ cat <(echo "Header") <(cat dados.txt) <(echo "Footer")

# Process substitution com escrita
$ tee >(gzip > backup.gz) < arquivo.txt

4.6 tee — Lê stdin e Escreve em stdout e Arquivos

O tee resolve o problema de querer ver a saida na tela E salvar em arquivo ao mesmo tempo.

# Ver na tela e salvar em arquivo
$ ls -la | tee lista.txt

# Anexar em vez de sobrescrever
$ echo "nova linha" | tee -a log.txt

# Escrever em multiplos arquivos
$ comando | tee arquivo1.txt arquivo2.txt

# Usar com sudo (permite redirecionamento com privilegios)
$ echo "config" | sudo tee /etc/arquivo.conf

# Pipe para tee e depois para outro comando
$ comando | tee log.txt | grep "erro"

# Tee com process substitution
$ comando | tee >(grep "erro" > erros.log) >(grep "aviso" > avisos.log)

# Tee para /dev/null (apenas log)
$ comando | tee /dev/tty | gzip > saida.gz

📚 Volume 5 — Gerenciamento de Processos

Entender, monitorar e controlar processos é essencial para dominar Linux.

5.1 O que é um Processo?

Um processo é uma instância em execução de um programa. Cada processo tem:

  • PID (Process ID): identificador único
  • PPID (Parent PID): PID do processo pai
  • UID/GID: usuário e grupo dono
  • Espaço de memória: código, dados, heap, stack
  • Descritores de arquivo: stdin (0), stdout (1), stderr (2), e outros
  • Variáveis de ambiente
  • Diretório de trabalho
  • Estado: running, sleeping, stopped, zombie
┌─────────────────────────────────────────┐ │ PROCESSO (PID 1234) │ │ │ │ ┌─────────┐ ┌─────────┐ ┌─────────┐ │ │ │ Código │ │ Dados │ │ Heap │ │ │ │ (text) │ │ (data) │ │ │ │ │ └─────────┘ └─────────┘ └─────────┘ │ │ ┌─────────┐ ┌─────────────────────┐ │ │ │ Stack │ │ Descritores (fd) │ │ │ │ │ │ 0=stdin, 1=stdout │ │ │ └─────────┘ │ 2=stderr, 3=arquivo │ │ │ └─────────────────────┘ │ │ ┌─────────┐ ┌─────────────────────┐ │ │ │ PID │ │ Variáveis de Amb. │ │ │ │ PPID │ │ PATH, HOME, USER │ │ │ │ UID/GID│ └─────────────────────┘ │ │ └─────────┘ │ └─────────────────────────────────────────┘

9.4 git — Version Control

O git é o sistema de controle de versao mais usado no mundo. Essencial para qualquer desenvolvedor.

9.4.1 Comandos Essenciais

# Inicializar repositorio
$ git init

# Clonar repositorio
$ git clone https://github.com/usuario/repo.git
$ git clone https://github.com/usuario/repo.git meu_repo

# Ver status
$ git status

# Adicionar arquivo ao stage
$ git add arquivo.txt
$ git add .                    # todos
$ git add -p                   # interativo (patch)

# Commit
$ git commit -m "Mensagem do commit"
$ git commit -am "Mensagem"    # add + commit (arquivos rastreados)

# Ver log
$ git log
$ git log --oneline
$ git log --graph --oneline --all
$ git log -p                   # com diff
$ git log --stat               # com estatisticas

# Ver diff
$ git diff                     # working vs staged
$ git diff --staged            # staged vs last commit
$ git diff HEAD~1              # diff com commit anterior

9.4.2 Branches

# Listar branches
$ git branch
$ git branch -a                # todas (incluindo remotas)
$ git branch -v                # com ultimo commit

# Criar branch
$ git branch nova-feature

# Mudar de branch
$ git checkout nova-feature
$ git switch nova-feature      # novo comando (git 2.23+)

# Criar e mudar
$ git checkout -b nova-feature
$ git switch -c nova-feature   # novo comando

# Merge
$ git checkout main
$ git merge nova-feature

# Rebase
$ git checkout nova-feature
$ git rebase main

# Deletar branch
$ git branch -d nova-feature   # merged
$ git branch -D nova-feature   # force delete

# Renomear branch
$ git branch -m novo-nome

9.4.3 Remoto

# Ver remotos
$ git remote -v

# Adicionar remoto
$ git remote add origin https://github.com/usuario/repo.git

# Fetch (baixa sem merge)
$ git fetch origin

# Pull (fetch + merge)
$ git pull origin main

# Push
$ git push origin main
$ git push -u origin main    # upstream

# Push de nova branch
$ git push -u origin nova-feature

# Deletar branch remota
$ git push origin --delete nova-feature

# Tags
$ git tag v1.0.0
$ git push origin v1.0.0
$ git push origin --tags

9.4.4 Desfazer e Corrigir

# Desfazer alteracoes nao staged
$ git checkout -- arquivo.txt
$ git restore arquivo.txt      # novo comando

# Unstage
$ git reset HEAD arquivo.txt
$ git restore --staged arquivo.txt  # novo comando

# Amend (alterar ultimo commit)
$ git commit --amend -m "Nova mensagem"
$ git commit --amend --no-edit   # sem alterar mensagem

# Reset
$ git reset --soft HEAD~1        # desfaz commit, mantem staged
$ git reset --mixed HEAD~1       # desfaz commit, mantem working
$ git reset --hard HEAD~1        # desfaz TUDO (CUIDADO!)

9.5 ssh — Secure Shell

O ssh permite acesso remoto seguro a servidores Linux.

9.5.1 Conexao Basica

# Conectar a servidor remoto
$ ssh usuario@servidor.com

# Conectar com porta especifica
$ ssh -p 2222 usuario@servidor.com

# Conectar com chave privada
$ ssh -i ~/.ssh/minha_chave usuario@servidor.com

# Executar comando remoto
$ ssh usuario@servidor.com "ls -la"

# Executar comando remoto com stdin local
$ cat arquivo.txt | ssh usuario@servidor.com "cat > /remoto/arquivo.txt"

9.5.2 Gerenciamento de Chaves

# Gerar par de chaves
$ ssh-keygen -t ed25519 -C "email@example.com"
$ ssh-keygen -t rsa -b 4096 -C "email@example.com"

# Copiar chave publica para servidor
$ ssh-copy-id usuario@servidor.com

# Ou manualmente:
$ cat ~/.ssh/id_ed25519.pub | ssh usuario@servidor.com "mkdir -p ~/.ssh && cat >> ~/.ssh/authorized_keys"

# Iniciar agente SSH
$ eval "$(ssh-agent -s)"
$ ssh-add ~/.ssh/id_ed25519

# Listar chaves no agente
$ ssh-add -l

# Remover chave do agente
$ ssh-add -d ~/.ssh/id_ed25519

# Remover todas as chaves
$ ssh-add -D

9.5.3 Configuracao SSH

# ~/.ssh/config

Host meu-servidor
    HostName servidor.com
    User usuario
    Port 2222
    IdentityFile ~/.ssh/minha_chave
    ServerAliveInterval 60
    ServerAliveCountMax 3

Host github
    HostName github.com
    User git
    IdentityFile ~/.ssh/github_key

# Uso:
$ ssh meu-servidor
$ git clone github:usuario/repo.git

9.5.4 Tunelamento (Port Forwarding)

# Local forward: porta local -> porta remota
$ ssh -L 8080:localhost:80 usuario@servidor.com
# Acesse localhost:8080 para chegar ao servidor:80

# Remote forward: porta remota -> porta local
$ ssh -R 9090:localhost:3000 usuario@servidor.com
# No servidor, acesse localhost:9090 para chegar ao seu :3000

# Dynamic forward (SOCKS proxy)
$ ssh -D 1080 usuario@servidor.com
# Configure navegador para usar SOCKS5 localhost:1080

# Background (nao trava terminal)
$ ssh -f -N -L 8080:localhost:80 usuario@servidor.com

# Gateway ports (permitir acesso externo ao tunnel)
$ ssh -L 0.0.0.0:8080:localhost:80 usuario@servidor.com

9.6 scp — Secure Copy

O scp copia arquivos entre hosts via SSH.

# Copiar local para remoto
$ scp arquivo.txt usuario@servidor:/home/usuario/

# Copiar remoto para local
$ scp usuario@servidor:/home/usuario/arquivo.txt ./

# Copiar diretorio recursivamente
$ scp -r diretorio/ usuario@servidor:/home/usuario/

# Copiar com porta especifica
$ scp -P 2222 arquivo.txt usuario@servidor:/home/usuario/

# Copiar com chave especifica
$ scp -i ~/.ssh/minha_chave arquivo.txt usuario@servidor:/home/usuario/

# Copiar preservando atributos
$ scp -p arquivo.txt usuario@servidor:/home/usuario/

# Copiar com compressao
$ scp -C arquivo.txt usuario@servidor:/home/usuario/

# Copiar com limite de banda (KB/s)
$ scp -l 1000 arquivo.txt usuario@servidor:/home/usuario/

# Copiar multiplos arquivos
$ scp arquivo1.txt arquivo2.txt usuario@servidor:/home/usuario/

# Copiar com verbose
$ scp -v arquivo.txt usuario@servidor:/home/usuario/
💡 Preferencia moderna: rsync

Para transferencias frequentes ou grandes, rsync eh geralmente preferido ao scp porque resume transferencias interrompidas, sincroniza apenas diferencas, e eh mais eficiente:

$ rsync -avz --progress arquivo.txt usuario@servidor:/home/usuario/
$ rsync -avz --delete diretorio/ usuario@servidor:/home/usuario/diretorio/
