7.1 apt — Advanced Package Tool

O apt é a interface de alto nível para gerenciamento de pacotes em sistemas Debian/Ubuntu. Ele lida com dependências, repositórios, e resolução de conflitos.

7.1.1 Atualizar Informações de Pacotes

# Atualizar lista de pacotes disponíveis
$ sudo apt update

# Atualizar com debug
$ sudo apt update -o Debug::Acquire::http=true

# Atualizar apenas um componente
$ sudo apt update -o Dir::Etc::sourcelist="sources.list.d/meu_repo.list"

7.1.2 Instalar Pacotes

# Instalar pacote
$ sudo apt install nginx

# Instalar múltiplos pacotes
$ sudo apt install nginx mysql-server php

# Instalar sem recomendações
$ sudo apt install --no-install-recommends nginx

# Instalar versão específica
$ sudo apt install nginx=1.18.0-0ubuntu1

# Simular instalação (não instala, apenas mostra o que faria)
$ sudo apt install --simulate nginx
$ sudo apt install -s nginx

# Baixar apenas (não instalar)
$ sudo apt install --download-only nginx
# Pacotes vão para /var/cache/apt/archives/

# Instalar de arquivo .deb
$ sudo apt install ./pacote.deb

# Instalar com confirmação automática
$ sudo apt install -y nginx

# Instalar com reinstalação completa
$ sudo apt install --reinstall nginx

7.1.3 Remover Pacotes

# Remover pacote (mantém configurações)
$ sudo apt remove nginx

# Remover completamente (incluindo configurações)
$ sudo apt purge nginx
$ sudo apt remove --purge nginx

# Remover pacotes órfãos (dependências não mais necessárias)
$ sudo apt autoremove

# Remover pacotes órfãos com confirmação
$ sudo apt autoremove -y

# Remover pacotes baixados do cache
$ sudo apt clean

# Remover pacotes antigos do cache
$ sudo apt autoclean

7.1.4 Atualizar Sistema

# Atualizar pacotes instalados (sem remover nada)
$ sudo apt upgrade

# Atualizar com remoção de pacotes obsoletos
$ sudo apt full-upgrade
# (antigamente: apt dist-upgrade)

# Atualização não-interativa
$ sudo apt upgrade -y

# Atualizar apenas pacotes de segurança
$ sudo apt upgrade -y -o APT::Get::Upgrade-Allow-New=false

# Simular atualização
$ sudo apt upgrade --simulate

7.1.5 Buscar e Informações

# Buscar pacote por nome/descrição
$ apt search nginx

# Buscar com regex
$ apt search ^nginx

# Informações detalhadas do pacote
$ apt show nginx

# Listar dependências
$ apt depends nginx

# Listar dependências reversas (quem depende deste)
$ apt rdepends nginx

# Listar arquivos de um pacote instalado
$ dpkg -L nginx

# Listar pacotes instalados
$ apt list --installed

# Verificar se pacote está instalado
$ apt list --installed | grep nginx

# Verificar versão instalada
$ apt list --installed nginx

# Política de versões (repositórios)
$ apt policy nginx

# Cache de pacotes
$ apt cache show nginx

7.1.6 Gerenciar Repositórios

# Listar repositórios configurados
$ cat /etc/apt/sources.list
$ ls /etc/apt/sources.list.d/

# Adicionar repositório (Debian/Ubuntu)
$ sudo add-apt-repository ppa:usuario/repo

# Adicionar repositório manualmente
$ echo "deb http://repo.example.com/ubuntu jammy main" | sudo tee /etc/apt/sources.list.d/meu_repo.list

# Adicionar chave GPG
$ wget -qO - https://repo.example.com/key.gpg | sudo apt-key add -
# ou (moderno):
$ sudo wget -qO /usr/share/keyrings/meu_repo.gpg https://repo.example.com/key.gpg

# Remover repositório
$ sudo add-apt-repository --remove ppa:usuario/repo
$ sudo rm /etc/apt/sources.list.d/meu_repo.list

# Editar sources.list com editor seguro
$ sudo apt edit-sources

7.2 dpkg — Debian Package Manager

O dpkg é a ferramenta de baixo nível para manipular pacotes .deb. Não resolve dependências automaticamente — use apt para isso.

7.2.1 Instalar e Remover

# Instalar pacote .deb
$ sudo dpkg -i pacote.deb

# Instalar com resolução de dependências (use apt)
$ sudo apt install ./pacote.deb

# Remover pacote
$ sudo dpkg -r pacote

# Remover completamente
$ sudo dpkg -P pacote

# Reconfigurar pacote
$ sudo dpkg-reconfigure pacote

# Reconfigurar todos os pacotes
$ sudo dpkg-reconfigure -a

7.2.2 Consultar Informações

# Listar pacotes instalados
$ dpkg -l
$ dpkg -l | grep nginx

# Informações de um pacote
$ dpkg -s nginx

# Listar arquivos de um pacote
$ dpkg -L nginx

# Descobrir qual pacote possui um arquivo
$ dpkg -S /usr/bin/nginx
$ dpkg -S /etc/nginx/nginx.conf

# Status de um pacote
$ dpkg -s nginx | grep Status

# Listar arquivos de configuração
$ dpkg -L nginx | grep /etc/

# Verificar integridade de pacotes
$ dpkg -V
$ dpkg -V nginx

7.2.3 Depuração e Reparo

# Verificar pacotes quebrados
$ dpkg -C

# Reconfigurar pacotes desconfigurados
$ sudo dpkg --configure -a

# Forçar reinstalação
$ sudo dpkg -i --force-all pacote.deb

# Ignorar conflitos
$ sudo dpkg -i --force-overwrite pacote.deb

# Remover forçado
$ sudo dpkg -r --force-depends pacote

# Limpar estado de pacote
$ sudo dpkg --clear-avail

# Atualizar banco de dados
$ sudo dpkg --update-avail

7.3 apt-cache — Query the APT Cache

# Buscar pacotes
$ apt-cache search nginx

# Informações detalhadas
$ apt-cache show nginx

# Dependências
$ apt-cache depends nginx

# Dependências reversas
$ apt-cache rdepends nginx

# Estatísticas do cache
$ apt-cache stats

# Política de versão
$ apt-cache policy nginx

# Listar pacotes disponíveis
$ apt-cache pkgnames
$ apt-cache pkgnames nginx

💡 Recomendação

Use apt para uso interativo (mais amigável). Use apt-get em scripts (mais estável, output mais previsível). apt é um wrapper que chama apt-get e apt-cache por baixo.

🏋️ Exercícios — Volume 7

  1. Execute apt update e observe a saída. Quantos pacotes podem ser atualizados?
  2. Use apt search para encontrar 3 pacotes relacionados a "editor de texto".
  3. Use apt show para ver informações detalhadas de um pacote instalado. Identifique: versão, tamanho, dependências, descrição.
  4. Use dpkg -l e conte quantos pacotes estão instalados no seu sistema.
  5. Use dpkg -S para descobrir qual pacote fornece o comando ls.
  6. Use apt list --installed e filtre apenas pacotes que começam com "lib".
  7. Verifique apt policy de um pacote e interprete: qual versão está instalada? Qual está disponível? De qual repositório?
  8. Execute apt autoremove --simulate e veja quais pacotes seriam removidos. Execute de verdade se fizer sentido.
  9. Use apt-cache depends e apt-cache rdepends para um pacote e trace a árvore de dependências.
  10. Crie um script que verifique se há atualizações pendentes e envie um alerta se houver.
