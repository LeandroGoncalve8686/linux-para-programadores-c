📚 Volume 2 — Terminal: Comandos Essenciais

Os comandos que você usará todos os dias. Cada comando com dezenas de exemplos, flags e casos de uso reais.

2.1 pwd — Print Working Directory

O comando pwd exibe o caminho absoluto do diretório atual. Simples, mas fundamental para saber onde você está no sistema de arquivos.

$ pwd
/home/usuario/projetos/linux

# pwd lógico (respeita links simbólicos)
$ pwd -L
/home/usuario/link_para_projetos

# pwd físico (resolve links simbólicos)
$ pwd -P
/home/usuario/projetos/linux
💡 Dica

Seu prompt (PS1) geralmente já mostra o diretório atual, mas pwd é útil em scripts onde você precisa capturar o caminho em uma variável: DIR_ATUAL=$(pwd)

2.2 ls — List Directory Contents

O comando ls é um dos mais usados no terminal. Lista arquivos e diretórios com diversas opções de formatação.

2.2.1 Opções Essenciais

# Listagem básica
$ ls
arquivo1.txt  diretorio/  script.sh

# Listagem detalhada (long format)
$ ls -l
drwxr-xr-x 2 usuario grupo 4096 Jun 14 10:00 diretorio
-rw-r--r-- 1 usuario grupo 1234 Jun 14 09:30 arquivo.txt
-rwxr-xr-x 1 usuario grupo  512 Jun 14 08:00 script.sh

# Listagem com tamanhos human-readable
$ ls -lh
-rw-r--r-- 1 usuario grupo 1.2K Jun 14 09:30 arquivo.txt
-rw-r--r-- 1 usuario grupo 2.4M Jun 14 09:45 video.mp4

# Mostrar arquivos ocultos (começam com .)
$ ls -la
.  ..  .bashrc  .profile  arquivo.txt  diretorio/

# Ordenar por data de modificação (mais recente primeiro)
$ ls -lt

# Ordenar por data de acesso
$ ls -ltu

# Ordenar por tamanho (maior primeiro)
$ ls -lhS

# Ordenar reverso
$ ls -ltr   # mais antigo primeiro

# Mostrar tipo de arquivo (indicador no final)
$ ls -F
arquivo.txt  diretorio/  link_simbolico@  executavel*

# Listar recursivamente
$ ls -R

# Listar com inode number
$ ls -i
123456 arquivo.txt

# Listar apenas diretórios
$ ls -d */

# Listar com cores (auto-detecta terminal)
$ ls --color=auto

# Mostrar UID/GID numéricos
$ ls -ln

# Mostrar tempo completo
$ ls -l --time-style=full-iso

2.2.2 Combinações Poderosas

# Listar todos os arquivos, ordenados por tamanho, com tamanhos legíveis
$ ls -lahS

# Listar arquivos modificados nas últimas 24 horas
$ ls -lt --time-style=full-iso | head -10

# Listar apenas arquivos executáveis
$ ls -l | grep '^-..x'

# Contar número de arquivos em um diretório
$ ls -1 | wc -l

# Listar arquivos com extensão .txt
$ ls -l *.txt

# Listar arquivos que NÃO são .txt
$ ls -l | grep -v '\.txt$'
⚠️ Cuidado com ls em scripts

Nunca use ls para processar arquivos em scripts (especialmente nomes com espaços). Use find com -print0 ou globbing seguro. ls é para visualização humana, não para processamento automatizado.

2.3 cd — Change Directory

O comando cd muda o diretório atual. É um comando interno do shell (built-in), não um programa externo.

# Ir para um diretório específico
$ cd /home/usuario/projetos

# Ir para o diretório home
$ cd
$ cd ~
$ cd ~usuario

# Ir para o diretório anterior
$ cd -

# Subir um nível
$ cd ..

# Subir dois níveis
$ cd ../..

# Ir para um diretório relativo
$ cd projetos/linux

# Ir para diretório absoluto
$ cd /var/log

# Usar tab completion
$ cd /u/l/b<TAB>  →  /usr/local/bin

# Usar variável de ambiente
$ cd $HOME
$ cd $PWD

# CDPATH — caminhos de busca para cd
$ export CDPATH=.:~/projetos:/var/www
$ cd site   # procura em ., ~/projetos, /var/www

2.4 mkdir — Make Directory

Cria diretórios. Parece simples, mas tem opções poderosas.

# Criar um diretório
$ mkdir novo_diretorio

# Criar múltiplos diretórios
$ mkdir dir1 dir2 dir3

# Criar diretórios aninhados (parents)
$ mkdir -p projetos/2024/junho/relatorios
# Cria toda a hierarquia se não existir

# Criar com permissões específicas
$ mkdir -m 755 diretorio_publico

# Criar diretório e verificar
$ mkdir -v novo_dir
mkdir: created directory 'novo_dir'

# Criar estrutura de projeto completa
$ mkdir -p meu_projeto/{src,include,lib,tests,docs,bin}
# Cria: meu_projeto/src, meu_projeto/include, etc.

# Criar diretório com data atual no nome
$ mkdir backup_$(date +%Y-%m-%d)
# Resultado: backup_2024-06-14
💡 Brace Expansion

O bash suporta brace expansion para criar múltiplos itens de forma concisa:

$ mkdir -p projeto/{src,tests,docs}/{2023,2024}
# Cria: projeto/src/2023, projeto/src/2024, projeto/tests/2023, ...

2.5 rmdir — Remove Directory

Remove diretórios vazios. Para diretórios com conteúdo, use rm -r (com cuidado!).

# Remover diretório vazio
$ rmdir diretorio_vazio

# Remover múltiplos diretórios vazios
$ rmdir dir1 dir2 dir3

# Remover diretório e seus pais se ficarem vazios
$ rmdir -p projeto/2024/junho
# Remove junho/, se 2024/ ficar vazio remove também, etc.

# Remover diretórios vazios recursivamente
$ find . -type d -empty -delete

# Verificar antes de remover
$ rmdir -v diretorio
rmdir: removing directory, 'diretorio'
🚨 rmdir vs. rm -r

rmdir só remove diretórios vazios — é seguro. rm -r remove diretórios e TODO o conteúdo. Use rm -ri (interativo) ou rm -rI (pergunta uma vez para muitos arquivos) para evitar acidentes.

2.6 touch — Change File Timestamps

O touch é usado para criar arquivos vazios ou atualizar timestamps. É mais poderoso do que parece.

# Criar arquivo vazio
$ touch novo_arquivo.txt

# Atualizar timestamp de acesso e modificação para agora
$ touch arquivo_existente.txt

# Atualizar apenas timestamp de acesso
$ touch -a arquivo.txt

# Atualizar apenas timestamp de modificação
$ touch -m arquivo.txt

# Não criar arquivo se não existir
$ touch -c arquivo_inexistente.txt

# Usar timestamp de outro arquivo como referência
$ touch -r arquivo_referencia.txt arquivo_alvo.txt

# Definir timestamp específico
$ touch -t 202406141200.00 arquivo.txt
# Formato: [[CC]YY]MMDDhhmm[.ss]
# 2024-06-14 12:00:00

# Definir data com string legível
$ touch -d "2024-06-14 12:00:00" arquivo.txt
$ touch -d "yesterday" arquivo.txt
$ touch -d "2 days ago" arquivo.txt
$ touch -d "next Monday" arquivo.txt

# Criar múltiplos arquivos
$ touch arquivo{1..10}.txt
# Cria: arquivo1.txt, arquivo2.txt, ..., arquivo10.txt

# Criar arquivo com espaço no nome (use aspas)
$ touch "meu arquivo.txt"

# Criar arquivo oculto
$ touch .config

2.6.1 Timestamps no Linux

TimestampSignificadoComando

2.7 cp — Copy Files and Directories

Copia arquivos e diretórios. Um dos comandos mais perigosos se usado sem atenção.

# Copiar arquivo
$ cp origem.txt destino.txt

# Copiar para diretório
$ cp arquivo.txt /backup/

# Copiar múltiplos arquivos
$ cp arquivo1.txt arquivo2.txt /destino/

# Copiar com preservação de atributos
$ cp -p arquivo.txt backup/
# Preserva: modo, ownership, timestamps

# Copiar recursivamente (diretórios)
$ cp -r diretorio/ /backup/

# Copiar recursivamente com preservação completa
$ cp -a diretorio/ /backup/
# Equivalente a -dR --preserve=all

# Copiar com verificação de sobrescrita (interativo)
$ cp -i arquivo.txt /destino/

# Forçar sobrescrita sem perguntar
$ cp -f arquivo.txt /destino/

# Copiar apenas se o arquivo for mais novo
$ cp -u arquivo.txt /destino/

# Copiar com barra de progresso (pv)
$ cp arquivo_grande.iso /destino/ & pv -pteba arquivo_grande.iso > /destino/arquivo_grande.iso

# Copiar com backup automático
$ cp --backup=numbered arquivo.txt /destino/
# Cria: arquivo.txt.~1~, arquivo.txt.~2~, etc.

# Copiar links simbólicos como links (não seguir)
$ cp -d link_simbolico /destino/

# Criar link hard em vez de copiar
$ cp -l arquivo.txt /destino/

# Copiar com sparse (arquivos esparsos)
$ cp --sparse=always arquivo.img /destino/
🚨 Cuidado com cp e diretórios

cp -r copia recursivamente. Se o destino já existe e é um diretório, o conteúdo é copiado dentro dele. Se não existe, cria o diretório. Verifique sempre com ls antes.

2.8 mv — Move/Rename Files

Move ou renomeia arquivos e diretórios. No Linux, renomear e mover são a mesma operação (ambos alteram o path).

# Renomear arquivo
$ mv antigo.txt novo.txt

# Mover arquivo para diretório
$ mv arquivo.txt /destino/

# Mover múltiplos arquivos
$ mv *.txt /backup/

# Mover e sobrescrever sem perguntar
$ mv -f arquivo.txt /destino/

# Mover com confirmação (interativo)
$ mv -i arquivo.txt /destino/

# Mover apenas se o destino for mais antigo
$ mv -u arquivo.txt /destino/

# Mover com backup
$ mv --backup=numbered arquivo.txt /destino/

# Mover diretório inteiro
$ mv diretorio/ /novo/local/

# Renomear com substituição de padrão (bash)
$ for f in *.txt; do mv "$f" "${f%.txt}.bak"; done

# Renomear múltiplos arquivos com rename (perl)
$ rename 's/\.txt$/.bak/' *.txt

# Mover arquivos modificados hoje
$ find . -mtime -1 -type f -exec mv {} /backup/hoje/ \;
💡 mv é atômico para arquivos no mesmo filesystem

Quando mv move um arquivo dentro do mesmo filesystem, ele apenas atualiza os metadados no inode — os dados do arquivo não são copiados. Isso é instantâneo e atômico, mesmo para arquivos gigantes.

2.9 rm — Remove Files

Remove arquivos. PERMANENTEMENTE. Não há lixeira no terminal. Use com extrema cautela.

# Remover arquivo
$ rm arquivo.txt

# Remover múltiplos arquivos
$ rm arquivo1.txt arquivo2.txt

# Remover com confirmação
$ rm -i arquivo.txt

# Remover sem confirmação (padrão para root!)
$ rm -f arquivo.txt

# Remover diretório e conteúdo recursivamente
$ rm -r diretorio/

# Remover diretório vazio
$ rm -d diretorio_vazio/
# Equivalente a rmdir

# Remover recursivamente e forçado (MUITO CUIDADO!)
$ rm -rf diretorio/

# Remover arquivos com nomes estranhos
$ rm -- -arquivo_com_hifen_no_inicio
$ rm ./-arquivo_com_hifen

# Remover exceto um arquivo
$ rm !(arquivo_importante.txt)
# Requer: shopt -s extglob

# Remover arquivos antigos
$ find /tmp -type f -mtime +7 -delete

# Remover com log
$ rm -v arquivo.txt
removed 'arquivo.txt'

# Simular remoção (não remove, apenas mostra o que faria)
# Use echo ou rm -i combinado com find
$ find . -name "*.tmp" -exec echo rm {} \;
🚨 O Comando Mais Perigoso do Linux

rm -rf / apaga TODO o sistema. Algumas distribuições modernas têm proteção (rm --preserve-root é padrão), mas rm -rf /* ou rm -rf / --no-preserve-root ainda funcionam. Sempre verifique o caminho antes de executar rm -rf.

Regra: Nunca use rm -rf com variáveis não verificadas ou com wildcards sem testar primeiro.

2.10 cat — Concatenate and Print

O cat concatena e exibe arquivos. É frequentemente usado para visualizar arquivos pequenos, mas tem usos mais avançados.

# Exibir conteúdo de arquivo
$ cat arquivo.txt

# Concatenar múltiplos arquivos
$ cat arquivo1.txt arquivo2.txt > combinado.txt

# Concatenar com numeração de linhas
$ cat -n arquivo.txt

# Numeração apenas para linhas não-vazias
$ cat -b arquivo.txt

# Mostrar caracteres especiais (tabs, fim de linha)
$ cat -A arquivo.txt
# ^I = tab, $ = fim de linha

# Suprimir linhas repetidas em branco
$ cat -s arquivo.txt

# Criar arquivo com heredoc
$ cat > arquivo.txt << 'EOF'
Linha 1
Linha 2
Linha 3
EOF

# Criar arquivo com conteúdo inline
$ cat > config.ini < arquivo.txt
# Equivalente a: > arquivo.txt
⚠️ Useless Use of Cat (UUOC)

Usar cat arquivo | comando quando comando arquivo funciona é considerado um anti-pattern. O cat cria um processo desnecessário. Prefira grep "padrao" arquivo.txt em vez de cat arquivo.txt | grep "padrao".

2.11 tac — Concatenate and Print in Reverse

O tac é o inverso do cat: exibe o arquivo de trás para frente (por linhas).

# Exibir arquivo invertido (linhas)
$ tac arquivo.txt

# Usar separador customizado
$ tac -s "---" arquivo.txt

# Processar logs em ordem inversa (mais recente primeiro)
$ tac /var/log/syslog | head -20

# Combinar com grep para buscar do final
$ tac arquivo.log | grep "ERRO" | head -5

2.12 nl — Number Lines

O nl numera linhas de arquivo com mais controle que cat -n.

# Numerar todas as linhas
$ nl arquivo.txt

# Numerar apenas linhas não-vazias (padrão)
$ nl -b t arquivo.txt

# Numerar todas as linhas
$ nl -b a arquivo.txt

# Numerar apenas linhas com texto
$ nl -b n arquivo.txt

# Definir formato de numeração
$ nl -n rz arquivo.txt    # zeros à esquerda
$ nl -n ln arquivo.txt    # alinhado à esquerda
$ nl -n rn arquivo.txt    # alinhado à direita (padrão)

# Definir largura do número
$ nl -w 5 arquivo.txt

# Definir string de separação
$ nl -s ". " arquivo.txt
# 1. Linha um
# 2. Linha dois

# Usar delimitador de seção lógica
$ nl -d "@@" arquivo.txt

2.13 less — Paginator Avançado

O less é um paginador que permite navegar por arquivos grandes de forma interativa. É o sucessor do more e muito mais poderoso.

# Visualizar arquivo
$ less arquivo.txt

# Visualizar com numeração de linhas
$ less -N arquivo.txt

# Visualizar com highlight de busca
$ less -g arquivo.txt

# Ignorar case na busca
$ less -I arquivo.txt

# Visualizar múltiplos arquivos
$ less arquivo1.txt arquivo2.txt
# :n → próximo arquivo
# :p → arquivo anterior

# Visualizar saída de comando
$ dmesg | less
$ ps aux | less

# Buscar dentro do less
# /padrao    → buscar para frente
# ?padrao    → buscar para trás
# n          → próxima ocorrência
# N          → ocorrência anterior

# Navegação no less
# Space/PageDown → página abaixo
# b/PageUp       → página acima
# j/↓            → uma linha abaixo
# k/↑            → uma linha acima
# g              → início do arquivo
# G              → fim do arquivo
# 50g            → linha 50
# %50            → 50% do arquivo
# q              → sair

# Seguir arquivo em tempo real (como tail -f)
$ less +F arquivo.log
# Ctrl+C para pausar, F para continuar

# Editar arquivo diretamente do less
# v → abre no editor padrão ($VISUAL ou $EDITOR)
💡 less é mais que more

less permite navegar para trás (ao contrário de more), buscar, marcar posições, e visualizar arquivos sem carregá-los completamente na memória. Para arquivos grandes, less é essencial.

2.14 more — Paginador Simples

O more é um paginador básico. Na maioria dos casos, prefira less.

# Visualizar arquivo
$ more arquivo.txt

# Definir número de linhas por tela
$ more -10 arquivo.txt

# Desabilitar scroll
$ more -d arquivo.txt

# Navegação no more
# Space → próxima página
# Enter → próxima linha
# b     → página anterior (nem sempre funciona)
# /padrao → buscar
# q     → sair

2.15 head — Output First Part

Exibe as primeiras linhas de um arquivo. Útil para pré-visualizar arquivos grandes.

# Primeiras 10 linhas (padrão)
$ head arquivo.txt

# Primeiras 20 linhas
$ head -n 20 arquivo.txt
$ head -20 arquivo.txt

# Primeiros 100 bytes
$ head -c 100 arquivo.txt

# Primeiras linhas de múltiplos arquivos
$ head arquivo1.txt arquivo2.txt

# Combinar com pipe
$ cat /var/log/syslog | head -50
# Melhor: head -50 /var/log/syslog

# Ver cabeçalho de arquivo CSV
$ head -1 dados.csv

# Ver primeiras linhas de todos os arquivos .log
$ head -n 5 *.log

2.16 tail — Output Last Part

Exibe as últimas linhas de um arquivo. Essencial para monitorar logs em tempo real.

# Últimas 10 linhas
$ tail arquivo.txt

# Últimas 20 linhas
$ tail -n 20 arquivo.txt

# Últimos 100 bytes
$ tail -c 100 arquivo.txt

# Seguir arquivo em tempo real (follow)
$ tail -f /var/log/syslog

# Seguir e reabrir se o arquivo for rotacionado
$ tail -F /var/log/syslog
# Útil para logs que são renomeados/compactados

# Combinar head e tail para extrair linhas do meio
$ head -n 100 arquivo.txt | tail -n 20
# Linhas 81-100

# Monitorar múltiplos arquivos
$ tail -f /var/log/syslog /var/log/auth.log

# Ver logs desde uma linha específica
$ tail -n +50 arquivo.txt
# Mostra a partir da linha 50

# Usar com grep para filtrar logs em tempo real
$ tail -f /var/log/syslog | grep "ERRO"
✅ tail -f é indispensável para sysadmins

Quando um serviço está com problemas, tail -f /var/log/servico.log permite ver os erros em tempo real. Combine com grep para filtrar apenas o que interessa.

2.17 file — Determine File Type

O file identifica o tipo de arquivo analisando seu conteúdo, não apenas a extensão.

# Identificar tipo de arquivo
$ file arquivo.txt
arquivo.txt: ASCII text

$ file programa
programa: ELF 64-bit LSB executable, x86-64, version 1 (SYSV)

$ file imagem.png
imagem.png: PNG image data, 800 x 600, 8-bit/color RGBA

# Identificar múltiplos arquivos
$ file *

# Identificar com mais detalhes
$ file -i arquivo.txt
arquivo.txt: text/plain; charset=us-ascii

# Ver magic number em hexadecimal
$ file -k arquivo_desconhecido

# Listar todos os tipos de arquivo conhecidos
$ file -l

# Identificar arquivo dentro de arquivo compactado
$ file -z arquivo.tar.gz

# Verificar arquivo de core dump
$ file core
💡 Como file funciona?

O file usa um banco de dados de "magic numbers" (bytes iniciais característicos) localizado em /usr/share/file/magic. Por exemplo, arquivos ELF começam com 0x7F 'E' 'L' 'F', arquivos PNG com 0x89 'P' 'N' 'G'.

O stat exibe informações detalhadas sobre arquivos — muito mais que ls -l.

2.19 tree — Display Directory Tree

O tree exibe a estrutura de diretórios em formato de árvore. Não vem instalado por padrão em todas as distribuições.

# Instalar tree
$ sudo apt install tree    # Debian/Ubuntu
$ sudo dnf install tree    # Fedora
$ sudo pacman -S tree      # Arch

# Exibir árvore do diretório atual
$ tree
.
├── diretorio1
│   ├── arquivo1.txt
│   └── arquivo2.txt
├── diretorio2
│   └── subdir
│       └── arquivo3.txt
└── arquivo_raiz.txt

# Limitar profundidade
$ tree -L 2

# Mostrar apenas diretórios
$ tree -d

# Mostrar arquivos com padrão específico
$ tree -P "*.c"

# Ignorar padrão
$ tree -I "node_modules|__pycache__"

# Mostrar tamanhos
$ tree -h

# Mostrar permissões
$ tree -p

# Mostrar inode numbers
$ tree -i

# Salvar árvore em arquivo
$ tree > estrutura.txt

# Colorir saída
$ tree -C

# Mostrar arquivos ocultos
$ tree -a

# Ordenar por versão (natural)
$ tree -v

# Usar caracteres ASCII (não Unicode)
$ tree -A
💡 tree como documentação

Use tree -L 2 -I 'node_modules|*.pyc' > ESTRUTURA.md para documentar a estrutura de um projeto em READMEs.

2.20 echo — Display a Line of Text

O echo exibe texto. Simples, mas essencial para scripts e testes.

# Exibir texto simples
$ echo "Hello, World!"

# Exibir sem nova linha no final
$ echo -n "Sem nova linha"

# Exibir com interpretação de escapes
$ echo -e "Linha 1
Linha 2	Tabulado"

# Escapes comuns
# 
  → nova linha
# 	  → tabulação
# \  → barra invertida
#   → beep (alert)
# 
  → carriage return
#   → backspace

# Exibir valor de variável
$ echo $HOME
$ echo $PATH
$ echo $?

# Exibir resultado de comando
$ echo "Hoje é $(date +%Y-%m-%d)"

# Criar arquivo com echo
$ echo "conteúdo" > arquivo.txt

# Anexar a arquivo
$ echo "nova linha" >> arquivo.txt

# Exibir com aspas simples (não expande variáveis)
$ echo '$HOME'
$HOME

# Exibir com aspas duplas (expande variáveis)
$ echo "$HOME"
/home/usuario

# Exibir asterisco literal
$ echo "*"
*
$ echo *
# (lista todos os arquivos do diretório!)

2.21 printf — Format and Print Data

O printf é mais poderoso que echo, com formatação tipo C. É um built-in do bash e também um comando externo.

# Formatação básica
$ printf "Hello, %s!
" "World"

# Múltiplos argumentos
$ printf "Nome: %s, Idade: %d
" "João" 25

# Formatação de números
$ printf "Decimal: %d
" 42
$ printf "Hex: %x
" 255
$ printf "Octal: %o
" 8
$ printf "Float: %.2f
" 3.14159

# Alinhamento
$ printf "|%-10s|%10s|
" "Esquerda" "Direita"
# |Esquerda  |    Direita|

# Preencher com zeros
$ printf "%010d
" 42
# 0000000042

# Formatação de data
$ printf "%(%Y-%m-%d %H:%M:%S)T
" -1
# 2024-06-14 12:00:00

# Criar tabela formatada
$ printf "%-15s %10s %10s
" "ARQUIVO" "TAMANHO" "PERMISSÕES"
$ printf "%-15s %10d %10s
" "arquivo.txt" 1234 "-rw-r--r--"

# Usar em loop
$ for f in *.txt; do printf "Processando: %s
" "$f"; done

# Imprimir sem nova linha
$ printf "Carregando"
$ printf "."
$ printf "."
$ printf ".
"
