📚 Volume 6 — Redes no Linux

Diagnosticar, configurar e entender redes é fundamental para sistemas e seguranca.

6.1 ip — Show/Manipulate Routing, Devices, Policy Routing and Tunnels

O ip é o comando moderno para gerenciamento de rede no Linux, substituindo ifconfig, route, arp, e outros.

6.1.1 Interfaces de Rede

# Listar todas as interfaces
$ ip link show

# Listar interfaces com estado
$ ip link show up

# Ver detalhes de uma interface
$ ip link show eth0

# Habilitar interface
$ sudo ip link set eth0 up

# Desabilitar interface
$ sudo ip link set eth0 down

# Alterar MTU
$ sudo ip link set eth0 mtu 9000

# Alterar MAC address
$ sudo ip link set eth0 address 00:11:22:33:44:55

# Listar interfaces de forma resumida
$ ip -s link

# Estatísticas de interface
$ ip -s link show eth0

6.1.2 Endereços IP

# Listar endereços IP
$ ip addr show
$ ip a

# Listar endereços de uma interface
$ ip addr show eth0

# Adicionar endereço IP
$ sudo ip addr add 192.168.1.100/24 dev eth0

# Remover endereço IP
$ sudo ip addr del 192.168.1.100/24 dev eth0

# Limpar todos os endereços de uma interface
$ sudo ip addr flush dev eth0

# Mostrar apenas IPv4
$ ip -4 addr show

# Mostrar apenas IPv6
$ ip -6 addr show

6.1.3 Rotas

# Mostrar tabela de rotas
$ ip route show
$ ip r

# Adicionar rota
$ sudo ip route add 10.0.0.0/8 via 192.168.1.1

# Adicionar rota padrão
$ sudo ip route add default via 192.168.1.1

# Remover rota
$ sudo ip route del 10.0.0.0/8

# Mostrar rota para destino específico
$ ip route get 8.8.8.8

# Mostrar tabela de rotas com detalhes
$ ip route show table main
$ ip route show table local

# Mostrar cache de rotas
$ ip route show cache

6.1.4 Vizinhos (ARP)

# Mostrar tabela ARP
$ ip neigh show
$ ip n

# Adicionar entrada ARP estática
$ sudo ip neigh add 192.168.1.1 lladdr 00:11:22:33:44:55 dev eth0

# Remover entrada
$ sudo ip neigh del 192.168.1.1 dev eth0

# Limpar cache ARP
$ sudo ip neigh flush all

6.2 ping — Send ICMP ECHO_REQUEST

O ping testa conectividade com outro host usando pacotes ICMP.

# Ping básico
$ ping google.com

# Ping com contagem limitada
$ ping -c 4 google.com

# Ping com intervalo específico (segundos)
$ ping -i 2 google.com

# Ping flood (requer root, cuidado!)
$ sudo ping -f google.com

# Ping com tamanho de pacote
$ ping -s 1400 google.com

# Ping com timeout
$ ping -w 10 google.com

# Ping com TTL específico
$ ping -t 64 google.com

# Ping com timestamp
$ ping -D google.com

# Ping para IPv6
$ ping6 google.com
$ ping -6 google.com

# Ping para broadcast
$ ping -b 192.168.1.255

6.3 ss — Investigate Sockets

O ss é o substituto moderno do netstat, mais rápido e informativo.

# Listar todas as conexões
$ ss

# Listar todas as sockets (TCP, UDP, UNIX)
$ ss -a

# Listar apenas sockets TCP
$ ss -t

# Listar apenas sockets UDP
$ ss -u

# Listar sockets em estado LISTEN
$ ss -l

# Listar sockets com processos
$ ss -tp

# Listar sockets com numeração (não resolve nomes)
$ ss -n

# Combinar opções
$ ss -tlnp   # TCP, listening, numeric, processes
$ ss -tunap  # TCP+UDP, all, numeric, processes

# Filtrar por estado
$ ss state established
$ ss state listening
$ ss state time-wait

# Filtrar por porta
$ ss -tlnp sport = :80
$ ss -tlnp sport = :ssh

# Filtrar por endereço
$ ss -tlnp src 192.168.1.1

# Mostrar estatísticas
$ ss -s

# Mostrar sockets UNIX
$ ss -x

# Mostrar sockets raw
$ ss -w

# Mostrar socket memory usage
$ ss -m

6.4 netstat — Network Statistics

O netstat é o comando legado para estatísticas de rede. Ainda útil, mas ss é preferido.

# Listar todas as conexões
$ netstat -a

# Listar conexões TCP
$ netstat -at

# Listar conexões UDP
$ netstat -au

# Listar sockets em listening
$ netstat -l

# Listar com processos
$ sudo netstat -tlnp

# Listar com nomes resolvidos
$ netstat -tln

# Estatísticas de protocolo
$ netstat -s

# Tabela de rotas
$ netstat -r
$ netstat -rn

# Estatísticas de interface
$ netstat -i

# Conexões ativas com timers
$ netstat -o

# Mostrar multicast
$ netstat -g

6.5 curl — Transfer Data from URLs

O curl é uma ferramenta poderosa para transferir dados com URLs. Suporta HTTP, HTTPS, FTP, SFTP, e muitos outros protocolos.

# GET simples
$ curl https://api.example.com/dados

# Salvar em arquivo
$ curl -o arquivo.json https://api.example.com/dados

# Salvar com nome do servidor
$ curl -O https://example.com/arquivo.zip

# Seguir redirects
$ curl -L https://bit.ly/xyz

# Mostrar headers
$ curl -I https://example.com

# Mostrar headers e corpo
$ curl -i https://example.com

# Headers customizados
$ curl -H "Authorization: Bearer token123" https://api.example.com

# POST com dados
$ curl -X POST -d "nome=Joao&idade=25" https://api.example.com/usuarios

# POST com JSON
$ curl -X POST \
  -H "Content-Type: application/json" \
  -d '{"nome":"Joao","idade":25}' \
  https://api.example.com/usuarios

# POST com arquivo
$ curl -X POST -d @dados.json https://api.example.com/usuarios

# PUT
$ curl -X PUT -d @atualizacao.json https://api.example.com/usuarios/1

# DELETE
$ curl -X DELETE https://api.example.com/usuarios/1

# Autenticação básica
$ curl -u usuario:senha https://api.example.com

# Autenticação com arquivo .netrc
$ curl --netrc https://api.example.com

# Ignorar certificado SSL (INSEGURO!)
$ curl -k https://self-signed.example.com

# Usar certificado específico
$ curl --cacert ca.crt https://example.com

# Verbose (debug)
$ curl -v https://example.com

# Verbose com headers completos
$ curl -vv https://example.com

# Silent (sem progresso)
$ curl -s https://example.com

# Silent mas mostra erros
$ curl -sS https://example.com

# Mostrar apenas código HTTP
$ curl -s -o /dev/null -w "%{http_code}" https://example.com

# Download com resume
$ curl -C - -o arquivo.zip https://example.com/arquivo.zip

# Limitar velocidade de download
$ curl --limit-rate 100K -o arquivo.zip https://example.com/arquivo.zip

# Timeout
$ curl --connect-timeout 10 --max-time 30 https://example.com

# Download paralelo (múltiplas conexões)
$ curl -Z -o arquivo.zip https://example.com/arquivo.zip

# FTP
$ curl ftp://ftp.example.com/arquivo.txt
$ curl -u usuario:senha ftp://ftp.example.com/arquivo.txt

# Upload FTP
$ curl -T arquivo.txt ftp://ftp.example.com/

# SFTP
$ curl -u usuario sftp://servidor.com/arquivo.txt

# SCP
$ curl -u usuario scp://servidor.com/arquivo.txt

6.5.1 curl Avançado

# Cookies
$ curl -c cookies.txt https://example.com/login
$ curl -b cookies.txt https://example.com/perfil

# Proxy
$ curl -x http://proxy:8080 https://example.com
$ curl -x socks5://proxy:1080 https://example.com

# User-Agent customizado
$ curl -A "Mozilla/5.0" https://example.com

# Referer
$ curl -e "https://google.com" https://example.com

# Enviar arquivo multipart/form-data
$ curl -F "arquivo=@foto.jpg" https://example.com/upload
$ curl -F "nome=Joao" -F "arquivo=@foto.jpg" https://example.com/upload

# WebSocket (com upgrade)
$ curl -i -N \
  -H "Connection: Upgrade" \
  -H "Upgrade: websocket" \
  https://example.com/ws

# Formatar JSON de saída (com jq)
$ curl -s https://api.example.com/dados | jq

6.6 wget — The Non-Interactive Network Downloader

O wget é especializado em downloads. Diferente de curl, ele é não-interativo e ideal para scripts de download.

# Download simples
$ wget https://example.com/arquivo.zip

# Download com nome customizado
$ wget -O meu_arquivo.zip https://example.com/arquivo.zip

# Download em background
$ wget -b https://example.com/arquivo_grande.zip

# Continuar download interrompido
$ wget -c https://example.com/arquivo_grande.zip

# Download recursivo (site inteiro)
$ wget -r https://example.com

# Download recursivo com limite de profundidade
$ wget -r -l 2 https://example.com

# Download apenas arquivos específicos
$ wget -r -A "*.pdf" https://example.com/documentos

# Rejeitar certos arquivos
$ wget -r -R "*.jpg,*.png" https://example.com

# Download com autenticação
$ wget --user=usuario --password=senha https://example.com/protegido

# Download via FTP
$ wget ftp://ftp.example.com/arquivo.txt

# Mirror de site
$ wget --mirror --convert-links --adjust-extension \
  --page-requisites --no-parent https://example.com/pagina

# Limitar velocidade
$ wget --limit-rate=200k https://example.com/arquivo.zip

# Tentativas
$ wget --tries=10 https://example.com/arquivo.zip

# Timeout
$ wget --timeout=30 https://example.com/arquivo.zip

# Spoof user-agent
$ wget --user-agent="Mozilla/5.0" https://example.com

# Download de lista de URLs
$ wget -i urls.txt

# Não sobrescrever arquivos existentes
$ wget -nc https://example.com/arquivo.zip

# Forçar sobrescrita
$ wget -N https://example.com/arquivo.zip

# Log de download
$ wget -o log.txt https://example.com/arquivo.zip

# Quiet mode
$ wget -q https://example.com/arquivo.zip

6.7 traceroute — Trace Route to Host

O traceroute mostra o caminho (saltos) que os pacotes fazem até chegar ao destino.

# Traceroute básico
$ traceroute google.com

# Traceroute com ICMP (ao invés de UDP)
$ traceroute -I google.com

# Traceroute com TCP
$ traceroute -T google.com

# Traceroute com porta específica
$ traceroute -p 80 google.com

# Limitar número de saltos
$ traceroute -m 15 google.com

# Timeout por sonda
$ traceroute -w 5 google.com

# Número de sondas por salto
$ traceroute -q 3 google.com

# Não resolver nomes (mais rápido)
$ traceroute -n google.com

# Traceroute IPv6
$ traceroute6 google.com

# Usar interface específica
$ traceroute -i eth0 google.com
💡 Interpretando traceroute

Cada linha representa um salto (router). Os três tempos são RTT de três sondas. * * * significa timeout (firewall bloqueando ICMP/UDP, ou router não responde). Saltos altos de RTT indicam congestionamento ou rota subótima.

6.8 dig — DNS Lookup Utility

O dig é a ferramenta mais completa para consultas DNS.

# Consulta A record
$ dig google.com

# Consulta específica
$ dig google.com A
$ dig google.com AAAA
$ dig google.com MX
$ dig google.com NS
$ dig google.com TXT
$ dig google.com CNAME
$ dig google.com SOA

# Consulta reversa (IP para nome)
$ dig -x 8.8.8.8

# Consultar servidor DNS específico
$ dig @8.8.8.8 google.com

# Consulta curta (apenas resposta)
$ dig +short google.com

# Consulta com trace completo
$ dig +trace google.com

# Mostrar apenas seção de resposta
$ dig +noall +answer google.com

# Consulta com timeout
$ dig +time=5 google.com

# Consulta com tentativas
$ dig +retry=3 google.com

# Não usar search domains
$ dig +nostdns google.com

# Consulta ANY (todos os registros)
$ dig google.com ANY

# Transferência de zona (requer permissão)
$ dig @ns.example.com example.com AXFR

# Consulta DNSSEC
$ dig +dnssec google.com

# Verificar assinatura DNSSEC
$ dig +sigchase google.com

# Consulta com IPv6
$ dig -6 google.com

6.8.1 Interpretando saída do dig

$ dig +noall +answer google.com
google.com.     285     IN      A       142.250.80.46

# google.com.  → nome consultado
# 285          → TTL (segundos)
# IN           → classe (Internet)
# A            → tipo de registro
# 142.250...   → valor

6.9 nslookup — Query Internet Name Servers

O nslookup é uma alternativa mais simples ao dig, embora considerado legado.

# Consulta básica
$ nslookup google.com

# Consulta tipo específico
$ nslookup -query=MX google.com

# Consultar servidor específico
$ nslookup google.com 8.8.8.8

# Modo interativo
$ nslookup
> server 8.8.8.8
> set type=MX
> google.com
> exit

# Consulta reversa
$ nslookup 8.8.8.8

🏋️ Exercícios — Volume 6

  1. Use ip addr show e identifique: seu IP local, máscara de rede, e MAC address.
  2. Use ip route show para encontrar seu gateway padrão. Use ip route get 8.8.8.8 e interprete a saída.
  3. Execute ping -c 10 google.com e analise: TTL, RTT médio, packet loss.
  4. Use ss -tlnp para listar todos os serviços escutando em portas TCP. Identifique 3 serviços.
  5. Use curl -I para ver os headers HTTP de um site. Identique: servidor, content-type, status code.
  6. Use curl para fazer um POST com JSON para httpbin.org/post e verifique a resposta.
  7. Use wget --mirror para fazer mirror de um site pequeno (use um site de teste).
  8. Execute traceroute google.com e conte quantos saltos até o destino. Identifique algum salto com latência anômala.
  9. Use dig +trace para ver a resolução completa de um domínio, desde os root servers.
  10. Use dig para consultar registros MX, NS, e TXT de um domínio.
  11. Use ss para encontrar todas as conexões estabelecidas do seu sistema. Identifique para onde elas vão.
  12. Crie um script que use curl para verificar se um site está online (HTTP 200) e logue o resultado.
