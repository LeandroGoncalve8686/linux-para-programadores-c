9.1 gcc — GNU Compiler Collection

O gcc é o compilador C padrão no Linux. Compila codigo C em executaveis, bibliotecas e objetos.

9.1.1 Compilacao Basica

# Compilar um arquivo C
$ gcc programa.c -o programa

# Compilar e executar
$ gcc programa.c -o programa && ./programa

# Compilar com warnings
$ gcc -Wall programa.c -o programa

# Compilar com warnings extras
$ gcc -Wall -Wextra programa.c -o programa

# Compilar com todas as warnings
$ gcc -Wall -Wextra -Wpedantic programa.c -o programa

# Compilar com debug info
$ gcc -g programa.c -o programa

# Compilar com otimizacao
$ gcc -O2 programa.c -o programa

# Compilar com otimizacao maxima
$ gcc -O3 programa.c -o programa

# Compilar sem otimizacao (para debug)
$ gcc -O0 -g programa.c -o programa

# Compilar com sanitizers (deteccao de erros em runtime)
$ gcc -fsanitize=address programa.c -o programa
$ gcc -fsanitize=undefined programa.c -o programa

9.1.3 Compilacao em Multiplos Passos

# 1. Preprocessar
$ gcc -E programa.c -o programa.i

# 2. Compilar para assembly
$ gcc -S programa.i -o programa.s

# 3. Montar (assembler)
$ gcc -c programa.s -o programa.o

# 4. Linkar
$ gcc programa.o -o programa

# Ou tudo de uma vez:
$ gcc programa.c -o programa

# Compilar multiplos arquivos
$ gcc -c main.c -o main.o
$ gcc -c utils.c -o utils.o
$ gcc main.o utils.o -o programa
Codigo C (.c) | v Preprocessador (cpp) → Codigo preprocessado (.i) | v Compilador (cc1) → Assembly (.s) | v Assembler (as) → Objeto (.o) | v Linker (ld) → Executavel

9.2 make — Build Automation

O make automatiza a compilacao, gerenciando dependencias entre arquivos.

9.2.1 Makefile Basico

# Makefile

CC = gcc
CFLAGS = -Wall -Wextra -g -O2
TARGET = programa
SRCS = main.c utils.c
OBJS = $(SRCS:.c=.o)

.PHONY: all clean

all: $(TARGET)

$(TARGET): $(OBJS)
	$(CC) $(OBJS) -o $(TARGET)

%.o: %.c
	$(CC) $(CFLAGS) -c $< -o $@

clean:
	rm -f $(OBJS) $(TARGET)

# Uso:
# make        → compila tudo
# make clean  → remove objetos e executavel
# make main.o → compila apenas main.c

9.2.2 Variaveis Automaticas do make

VariavelSignificado
$@Nome do alvo (target)
$<Primeira dependencia
$^Todas as dependencias
$?Dependencias mais novas que o alvo
$*Stem (parte do padrao %)

9.2.3 Makefile Avancado

# Makefile avancado

CC = gcc
CFLAGS = -Wall -Wextra -g -O2
LDFLAGS = -lm -lpthread
TARGET = programa
SRCDIR = src
OBJDIR = obj
BINDIR = bin

SRCS = $(wildcard $(SRCDIR)/*.c)
OBJS = $(patsubst $(SRCDIR)/%.c,$(OBJDIR)/%.o,$(SRCS))

.PHONY: all clean dirs debug release

all: dirs $(BINDIR)/$(TARGET)

dirs:
	mkdir -p $(OBJDIR) $(BINDIR)

$(BINDIR)/$(TARGET): $(OBJS)
	$(CC) $(OBJS) -o $@ $(LDFLAGS)

$(OBJDIR)/%.o: $(SRCDIR)/%.c
	$(CC) $(CFLAGS) -c $< -o $@

clean:
	rm -rf $(OBJDIR) $(BINDIR)

debug: CFLAGS = -Wall -Wextra -g -O0 -DDEBUG
debug: all

release: CFLAGS = -Wall -Wextra -O3 -DNDEBUG
release: all

# Uso:
# make          → build padrao
# make debug    → build com debug
# make release  → build otimizada
# make clean    → limpa tudo

9.3 gdb — GNU Debugger

O gdb é o depurador para C/C++. Permite inspecionar variaveis, breakpoints, stack trace, e muito mais.

9.3.1 Iniciar o gdb

# Compilar com debug info
$ gcc -g programa.c -o programa

# Iniciar gdb
$ gdb ./programa

# Iniciar e passar argumentos
$ gdb --args ./programa arg1 arg2

# Analisar core dump
$ gdb ./programa core

# Anexar a processo em execucao
$ gdb -p 1234

9.3.2 Comandos do gdb

9.4 git — Version Control

O git é o sistema de controle de versao mais usado no mundo. Essencial para qualquer desenvolvedor.

9.4.1 Comandos Essenciais

# Inicializar repositorio
$ git init

# Clonar repositorio
$ git clone https://github.com/usuario/repo.git
$ git clone https://github.com/usuario/repo.git meu_repo

# Ver status
$ git status

# Adicionar arquivo ao stage
$ git add arquivo.txt
$ git add .                    # todos
$ git add -p                   # interativo (patch)

# Commit
$ git commit -m "Mensagem do commit"
$ git commit -am "Mensagem"    # add + commit (arquivos rastreados)

# Ver log
$ git log
$ git log --oneline
$ git log --graph --oneline --all
$ git log -p                   # com diff
$ git log --stat               # com estatisticas

# Ver diff
$ git diff                     # working vs staged
$ git diff --staged            # staged vs last commit
$ git diff HEAD~1              # diff com commit anterior

9.4.2 Branches

# Listar branches
$ git branch
$ git branch -a                # todas (incluindo remotas)
$ git branch -v                # com ultimo commit

# Criar branch
$ git branch nova-feature

# Mudar de branch
$ git checkout nova-feature
$ git switch nova-feature      # novo comando (git 2.23+)

# Criar e mudar
$ git checkout -b nova-feature
$ git switch -c nova-feature   # novo comando

# Merge
$ git checkout main
$ git merge nova-feature

# Rebase
$ git checkout nova-feature
$ git rebase main

# Deletar branch
$ git branch -d nova-feature   # merged
$ git branch -D nova-feature   # force delete

# Renomear branch
$ git branch -m novo-nome

9.4.3 Remoto

# Ver remotos
$ git remote -v

# Adicionar remoto
$ git remote add origin https://github.com/usuario/repo.git

# Fetch (baixa sem merge)
$ git fetch origin

# Pull (fetch + merge)
$ git pull origin main

# Push
$ git push origin main
$ git push -u origin main    # upstream

# Push de nova branch
$ git push -u origin nova-feature

# Deletar branch remota
$ git push origin --delete nova-feature

# Tags
$ git tag v1.0.0
$ git push origin v1.0.0
$ git push origin --tags

9.4.4 Desfazer e Corrigir

# Desfazer alteracoes nao staged
$ git checkout -- arquivo.txt
$ git restore arquivo.txt      # novo comando

# Unstage
$ git reset HEAD arquivo.txt
$ git restore --staged arquivo.txt  # novo comando

# Amend (alterar ultimo commit)
$ git commit --amend -m "Nova mensagem"
$ git commit --amend --no-edit   # sem alterar mensagem

# Reset
$ git reset --soft HEAD~1        # desfaz commit, mantem staged
$ git reset --mixed HEAD~1       # desfaz commit, mantem working
$ git reset --hard HEAD~1        # desfaz TUDO (CUIDADO!)

# Stash
$ git stash
$ git stash push -m "mensagem"
$ git stash list
$ git stash pop                  # aplica e remove
$ git stash apply                # aplica sem remover
$ git stash drop                 # remove
$ git stash clear                # remove todos

# Cherry-pick
$ git cherry-pick abc1234

# Revert (cria commit que desfaz)
$ git revert abc1234

9.4.5 git Avancado

# Bisect (encontrar commit que introduziu bug)
$ git bisect start
$ git bisect bad                 # commit atual tem bug
$ git bisect good v1.0           # versao boa
# git testa commits intermediarios
$ git bisect good                # ou bad
$ git bisect reset               # termina

# Reflog (historico de todas as operacoes)
$ git reflog

# Blame (quem escreveu cada linha)
$ git blame arquivo.c

# Log com filtro
$ git log --author="Joao"
$ git log --since="2 weeks ago"
$ git log --grep="bugfix"

# Diff entre branches
$ git diff main..feature

# Submodules
$ git submodule add https://github.com/usuario/lib.git
$ git submodule update --init --recursive

# Worktree (multiplos working directories)
$ git worktree add ../repo-fix fix-branch

# Hooks
$ ls .git/hooks/
# pre-commit, post-commit, pre-push, etc.

9.5 ssh — Secure Shell

O ssh permite acesso remoto seguro a servidores Linux.

9.5.1 Conexao Basica

# Conectar a servidor remoto
$ ssh usuario@servidor.com

# Conectar com porta especifica
$ ssh -p 2222 usuario@servidor.com

# Conectar com chave privada
$ ssh -i ~/.ssh/minha_chave usuario@servidor.com

# Executar comando remoto
$ ssh usuario@servidor.com "ls -la"

# Executar comando remoto com stdin local
$ cat arquivo.txt | ssh usuario@servidor.com "cat > /remoto/arquivo.txt"

9.5.2 Gerenciamento de Chaves

# Gerar par de chaves
$ ssh-keygen -t ed25519 -C "email@example.com"
$ ssh-keygen -t rsa -b 4096 -C "email@example.com"

# Copiar chave publica para servidor
$ ssh-copy-id usuario@servidor.com

# Ou manualmente:
$ cat ~/.ssh/id_ed25519.pub | ssh usuario@servidor.com "mkdir -p ~/.ssh && cat >> ~/.ssh/authorized_keys"

# Iniciar agente SSH
$ eval "$(ssh-agent -s)"
$ ssh-add ~/.ssh/id_ed25519

# Listar chaves no agente
$ ssh-add -l

# Remover chave do agente
$ ssh-add -d ~/.ssh/id_ed25519

# Remover todas as chaves
$ ssh-add -D

9.5.3 Configuracao SSH

# ~/.ssh/config

Host meu-servidor
    HostName servidor.com
    User usuario
    Port 2222
    IdentityFile ~/.ssh/minha_chave
    ServerAliveInterval 60
    ServerAliveCountMax 3

Host github
    HostName github.com
    User git
    IdentityFile ~/.ssh/github_key

# Uso:
$ ssh meu-servidor
$ git clone github:usuario/repo.git

9.5.4 Tunelamento (Port Forwarding)

# Local forward: porta local -> porta remota
$ ssh -L 8080:localhost:80 usuario@servidor.com
# Acesse localhost:8080 para chegar ao servidor:80

# Remote forward: porta remota -> porta local
$ ssh -R 9090:localhost:3000 usuario@servidor.com
# No servidor, acesse localhost:9090 para chegar ao seu :3000

# Dynamic forward (SOCKS proxy)
$ ssh -D 1080 usuario@servidor.com
# Configure navegador para usar SOCKS5 localhost:1080

# Background (nao trava terminal)
$ ssh -f -N -L 8080:localhost:80 usuario@servidor.com

# Gateway ports (permitir acesso externo ao tunnel)
$ ssh -L 0.0.0.0:8080:localhost:80 usuario@servidor.com

9.6 scp — Secure Copy

O scp copia arquivos entre hosts via SSH.

# Copiar local para remoto
$ scp arquivo.txt usuario@servidor:/home/usuario/

# Copiar remoto para local
$ scp usuario@servidor:/home/usuario/arquivo.txt ./

# Copiar diretorio recursivamente
$ scp -r diretorio/ usuario@servidor:/home/usuario/

# Copiar com porta especifica
$ scp -P 2222 arquivo.txt usuario@servidor:/home/usuario/

# Copiar com chave especifica
$ scp -i ~/.ssh/minha_chave arquivo.txt usuario@servidor:/home/usuario/

# Copiar preservando atributos
$ scp -p arquivo.txt usuario@servidor:/home/usuario/

# Copiar com compressao
$ scp -C arquivo.txt usuario@servidor:/home/usuario/

# Copiar com limite de banda (KB/s)
$ scp -l 1000 arquivo.txt usuario@servidor:/home/usuario/

# Copiar multiplos arquivos
$ scp arquivo1.txt arquivo2.txt usuario@servidor:/home/usuario/

# Copiar com verbose
$ scp -v arquivo.txt usuario@servidor:/home/usuario/
💡 Preferencia moderna: rsync

Para transferencias frequentes ou grandes, rsync eh geralmente preferido ao scp porque resume transferencias interrompidas, sincroniza apenas diferencas, e eh mais eficiente:

$ rsync -avz --progress arquivo.txt usuario@servidor:/home/usuario/
$ rsync -avz --delete diretorio/ usuario@servidor:/home/usuario/diretorio/
