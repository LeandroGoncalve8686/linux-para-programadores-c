📚 Volume 5 — Gerenciamento de Processos

Entender, monitorar e controlar processos é essencial para dominar Linux.

5.1 O que é um Processo?

Um processo é uma instância em execução de um programa. Cada processo tem:

  • PID (Process ID): identificador único
  • PPID (Parent PID): PID do processo pai
  • UID/GID: usuário e grupo dono
  • Espaço de memória: código, dados, heap, stack
  • Descritores de arquivo: stdin (0), stdout (1), stderr (2), e outros
  • Variáveis de ambiente
  • Diretório de trabalho
  • Estado: running, sleeping, stopped, zombie

5.2 ps — Report Process Status

O ps exibe informações sobre processos ativos. É um snapshot do momento da execução.

5.2.1 Uso Básico

# Processos do terminal atual
$ ps
  PID TTY          TIME CMD
 1234 pts/0    00:00:00 bash
 5678 pts/0    00:00:00 ps

# Todos os processos do usuário
$ ps -u $(whoami)

# Todos os processos (estilo BSD)
$ ps aux

# Todos os processos (estilo UNIX)
$ ps -ef

# Processos em formato customizado
$ ps -eo pid,ppid,cmd,%mem,%cpu

# Processos de um usuário específico
$ ps -u root

# Processos em uma árvore
$ ps axjf

5.2.2 Colunas do ps aux

5.2.4 ps Avançado

# Formato customizado com todas as colunas
$ ps -eo pid,ppid,uid,gid,tty,stat,priority,nice,pcpu,pmem,comm,args

# Processos ordenados por uso de CPU
$ ps aux --sort=-%cpu | head -10

# Processos ordenados por uso de memória
$ ps aux --sort=-%mem | head -10

# Processos com threads
$ ps -eLf | head -10

# Mostrar árvore de processos
$ ps axjf

# Mostrar apenas PIDs
$ ps -eo pid=

# Contar processos
$ ps aux | wc -l

# Encontrar processo por nome
$ ps aux | grep firefox
$ pgrep firefox

# Informações detalhadas de um processo
$ ps -p 1234 -o pid,ppid,cmd,etime,pcpu,pmem

5.3 top — Display Linux Processes

O top é um monitor de processos interativo em tempo real.

# Iniciar top
$ top

# Iniciar com delay de 1 segundo
$ top -d 1

# Mostrar apenas processos de um usuário
$ top -u usuario

# Mostrar processos de um PID específico
$ top -p 1234,5678

# Executar N ciclos e sair
$ top -n 5

# Salvar saída em batch mode
$ top -b -n 1 > processos.txt

5.5 pstree — Display Process Tree

O pstree mostra processos em formato de árvore, facilitando a visualização da hierarquia.

# Árvore de todos os processos
$ pstree

# Árvore com PIDs
$ pstree -p

# Árvore de um usuário específico
$ pstree usuario

# Árvore de um PID específico
$ pstree 1

# Mostrar argumentos completos
$ pstree -a

# Mostrar threads
$ pstree -t

# Mostrar PGIDs
$ pstree -g

# Destacar ancestral do processo atual
$ pstree -h

# Combinar opções
$ pstree -pahlu
systemd(1)─┬─accounts-daemon(567) ├─cron(589) ├─dbus-daemon(576) ├─sshd(612)───sshd(1234)───bash(1235)───pstree(5678) ├─systemd-journal(345) └─systemd-logind(456)

5.6 Jobs, Foreground e Background

O shell pode gerenciar múltiplos processos em uma sessão. Processos podem rodar em foreground (interativo) ou background (não-interativo).

5.6.1 Controle de Jobs

# Executar em background
$ sleep 60 &
[1] 1234

# Listar jobs
$ jobs
[1]+  Running                 sleep 60 &

# Trazer para foreground
$ fg %1

# Mandar para background
# Ctrl+Z para suspender, depois:
$ bg %1

# Suspender processo em foreground
# Ctrl+Z

# Matar um job
$ kill %1

# Trazer último job para foreground
$ fg

# Trazer job específico
$ fg %2

# Trazer job pelo nome
$ fg %sleep

5.6.2 Estados de Jobs

EstadoSignificado
RunningExecutando em background
StoppedSuspenso (Ctrl+Z)
DoneTerminado com sucesso
Done(code)Terminado com código de saída
TerminatedTerminado por sinal

5.6.3 nohup — Run Command Immune to Hangups

# Executar comando que sobrevive ao logout
$ nohup ./script.sh &
[1] 1234
nohup: ignoring input and appending output to 'nohup.out'

# Especificar arquivo de saída
$ nohup ./script.sh > meu_log.txt 2>&1 &

# Combinar com disown
$ ./script.sh &
$ disown %1
# O processo continua mesmo se o terminal fechar
💡 nohup vs. disown vs. screen/tmux
  • nohup: Executa comando ignorando SIGHUP. Simples, mas sem interação.
  • disown: Remove job da tabela de jobs do shell. O processo continua, mas sem controle fácil.
  • screen/tmux: Emuladores de terminal persistentes. Permitem reconectar posteriormente. Recomendado para sessões longas.

5.7 kill, killall, pkill — Enviar Sinais

Processos no Linux se comunicam via sinais. O comando kill envia sinais para processos.

5.7.1 Sinais Importantes

SinalNúmeroAçãoUso
SIGHUP1Hang upRecarregar config (daemon)
SIGINT2InterruptCtrl+C
SIGQUIT3QuitCtrl+\, gera core dump
SIGKILL9KillMatar imediatamente (não capturável)
SIGTERM15TerminateTerminar graciosamente (padrão)
SIGSTOP19StopCtrl+Z, não capturável
SIGCONT18ContinueContinuar processo parado
SIGUSR110User 1Uso definido pelo programa
SIGUSR212User 2Uso definido pelo programa

5.7.2 Uso do kill

# Enviar SIGTERM (padrão) para um PID
$ kill 1234

# Enviar SIGKILL (forçado)
$ kill -9 1234
$ kill -SIGKILL 1234

# Enviar SIGHUP (recarregar config)
$ kill -HUP 1234
$ kill -1 1234

# Enviar SIGINT (equivalente a Ctrl+C)
$ kill -INT 1234

# Enviar SIGSTOP (pausar)
$ kill -STOP 1234

# Enviar SIGCONT (continuar)
$ kill -CONT 1234

# Listar todos os sinais disponíveis
$ kill -l

# Verificar se processo existe
$ kill -0 1234
# Código 0 = processo existe; código != 0 = não existe

5.7.3 killall — Kill by Name

# Matar todos os processos com nome
$ killall firefox

# Matar com sinal específico
$ killall -9 firefox

# Listar processos que seriam mortos (simulação)
$ killall -i firefox

# Matar por usuário
$ sudo killall -u usuario firefox

# Matar processos mais antigos que N segundos
$ killall -o 1h firefox

# Matar processos mais novos que N segundos
$ killall -y 1m firefox

# Considerar nome exato
$ killall -e firefox

5.7.4 pkill — Kill by Pattern

# Matar por padrão no nome
$ pkill firefox

# Matar por padrão no comando completo
$ pkill -f "python script.py"

# Matar por usuário
$ pkill -u usuario

# Matar por terminal
$ pkill -t pts/0

# Enviar sinal específico
$ pkill -9 firefox

# Listar PIDs que seriam afetados
$ pgrep firefox

# Combinar critérios
$ pkill -u usuario -f "python"
⚠️ killall pode ser perigoso

killall mata TODOS os processos com aquele nome. Em alguns sistemas Unix (Solaris, AIX), killall mata TODOS os processos do sistema! Sempre use pgrep primeiro para verificar quais processos seriam afetados.

5.8 nice e renice — Ajustar Prioridade

A nice value (-20 a 19) determina a prioridade de um processo. Valores mais baixos = maior prioridade.

# Iniciar processo com prioridade baixa (nice alto)
$ nice -n 10 ./script.sh

# Iniciar com prioridade alta (requer root)
$ sudo nice -n -10 ./script.sh

# Alterar prioridade de processo em execução
$ renice -n 5 -p 1234

# Alterar prioridade de todos os processos de um usuário
$ sudo renice -n 5 -u usuario

# Alterar prioridade por nome
$ sudo renice -n -5 $(pgrep firefox)

# Ver nice value
$ ps -eo pid,comm,nice | grep firefox

-20MáximaProcessos críticos de tempo real
-10AltaCompilações, builds
0NormalPadrão
10BaixaBackups, indexação
19MínimaTarefas de manutenção

🏋️ Exercícios — Volume 5

  1. Execute ps aux e identifique: seu shell, o processo init (systemd), e 3 processos do sistema.
  2. Use ps -eo pid,ppid,stat,cmd e encontre um processo em estado Z (zombie). Se não houver, pesquise como criar um.
  3. Abra top, pressione 1 para ver CPUs individuais, M para ordenar por memória, e identifique o processo que mais consome recursos.
  4. Instale e use htop. Explore a interface, filtre por usuário, e mude a ordenação.
  5. Use pstree -p para ver a árvore completa do sistema. Encontre o seu shell e trace até o PID 1.
  6. Execute sleep 300, suspenda com Ctrl+Z, verifique com jobs, e continue em background com bg.
  7. Use nohup para executar um comando longo, feche o terminal, e verifique se o processo ainda está rodando em outro terminal.
  8. Use kill -l para listar todos os sinais. Envie SIGTERM e depois SIGKILL para um processo de teste. Qual a diferença?
  9. Use nice para iniciar um processo com prioridade 10 e renice para mudar para 5.
  10. Crie um script que monitore o uso de CPU de um processo específico a cada 2 segundos usando ps.
